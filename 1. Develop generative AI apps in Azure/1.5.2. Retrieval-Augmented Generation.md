# Retrieval-Augmented Generation (RAG)

## ¿Qué es?

**Retrieval-Augmented Generation (RAG)** es una técnica que mejora las respuestas de un modelo generativo al **recuperar información relevante de una fuente de datos externa** antes de generar la respuesta.

En lugar de depender únicamente del conocimiento adquirido durante el entrenamiento, el modelo utiliza información actualizada o privada como contexto. ([learn.microsoft.com](https://learn.microsoft.com/en-us/training/modules/optimize-generative-ai-model-performance/3-retrieval-augmented-generation?pivots=text))

---

# ¿Por qué usar RAG?

Los modelos tienen algunas limitaciones:

- Su conocimiento puede estar desactualizado.
- No conocen información privada de una organización.
- Pueden generar **alucinaciones** (información incorrecta o inventada).

RAG ayuda a resolver estos problemas proporcionando información relevante antes de que el modelo genere la respuesta.

---

# ¿Cómo funciona?

```text
Usuario
    │
    ▼
Pregunta
    │
    ▼
Búsqueda en la base de conocimiento
    │
    ▼
Recuperación de documentos relevantes
    │
    ▼
Prompt enriquecido con el contexto
    │
    ▼
Modelo generativo
    │
    ▼
Respuesta fundamentada
```

### Flujo

1. El usuario realiza una consulta.
2. Se buscan documentos relacionados.
3. Se recuperan los fragmentos más relevantes.
4. Esos fragmentos se agregan al prompt.
5. El modelo genera la respuesta utilizando ese contexto.

---

# Componentes de una solución RAG

## 1. Fuente de datos

Repositorio donde se almacena la información.

Ejemplos:

- Manuales.
- Documentación técnica.
- PDFs.
- Sitios web.
- Bases de conocimiento.
- Wikis internas.

---

## 2. Recuperación (Retrieval)

Busca los documentos o fragmentos más relevantes para responder la consulta.

Normalmente utiliza:

- Búsqueda semántica.
- Embeddings.
- Bases de datos vectoriales.

---

## 3. Generación (Generation)

El modelo recibe:

- La pregunta del usuario.
- El contexto recuperado.

Y genera una respuesta fundamentada en esa información.

---

# Ventajas

- Información más actualizada.
- Acceso a datos privados.
- Reduce las alucinaciones.
- No requiere volver a entrenar el modelo cuando cambian los documentos.
- Las respuestas pueden citar las fuentes utilizadas.

---

# ¿Cuándo usar RAG?

Utiliza RAG cuando necesites responder preguntas basadas en:

- Documentación empresarial.
- Manuales.
- Políticas internas.
- Información que cambia con frecuencia.
- Bases de conocimiento.

---

# RAG vs Fine-Tuning

| RAG | Fine-Tuning |
|------|-------------|
| Recupera información externa antes de responder. | Modifica el comportamiento del modelo mediante entrenamiento adicional. |
| La información puede actualizarse fácilmente. | Requiere volver a entrenar el modelo para incorporar nuevos conocimientos. |
| Ideal para conocimiento dinámico o privado. | Ideal para adaptar el estilo, formato o comportamiento del modelo. |
| Reduce alucinaciones utilizando fuentes reales. | No agrega conocimiento actualizado automáticamente. |

---

# Buenas prácticas

- Utilizar documentos de alta calidad.
- Mantener actualizada la base de conocimiento.
- Recuperar únicamente la información relevante.
- Evitar agregar demasiado contexto innecesario al prompt.
- Evaluar periódicamente la calidad de las respuestas.

---

# Resumen

- **Retrieval-Augmented Generation (RAG)** combina **búsqueda de información** y **generación de texto**.
- Antes de responder, recupera documentos relevantes y los incorpora como contexto.
- Sus principales beneficios son:
  - Acceso a información privada o actualizada.
  - Reducción de alucinaciones.
  - Respuestas fundamentadas en documentos reales.
- Es la mejor opción cuando el conocimiento cambia con frecuencia o proviene de fuentes externas.
