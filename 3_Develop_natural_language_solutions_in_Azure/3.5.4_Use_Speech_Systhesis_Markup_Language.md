# Use Speech Synthesis Markup Language (SSML)

## ¿Qué es SSML?
**Speech Synthesis Markup Language (SSML)** es un lenguaje basado en **XML** que permite controlar cómo se sintetiza la voz.

> Ofrece mucho más control que enviar solo texto plano.

## ¿Qué permite hacer?

- 😊 Definir un **estilo de voz** (cheerful, excited, etc.).
- ⏸️ Insertar pausas o silencios.
- 🔤 Especificar pronunciaciones (**phonemes**).
- 🎵 Ajustar la **prosodia** (tono, velocidad, timbre).
- 📅 Indicar cómo leer fechas, horas, teléfonos, etc. (`say-as`).
- 🎧 Insertar audio o grabaciones.

## Elementos comunes

| Elemento | Función |
|----------|---------|
| `<voice>` | Selecciona la voz. |
| `<mstts:express-as>` | Define el estilo de habla. |
| `<phoneme>` | Especifica la pronunciación. |
| `<break>` | Inserta una pausa. |
| `<say-as>` | Indica cómo leer un valor (fecha, teléfono, etc.). |

## Ejemplo conceptual

```xml
<speak>
    <voice name="en-US-AriaNeural">
        <mstts:express-as style="cheerful">
            Hello!
        </mstts:express-as>
    </voice>
</speak>
```

Resultado:
- Voz **Aria**.
- Tono **alegre (cheerful)**.

## Uso desde el SDK

En lugar de:

```python
SpeakTextAsync()
```

Se utiliza:

```python
SpeakSsmlAsync()
```

para enviar el documento SSML.

## Flujo

```text
SSML (XML)
      │
      ▼
SpeechSynthesizer
      │
      ▼
Azure Speech
      │
      ▼
Audio personalizado
```

## Ventajas frente a texto plano

| Texto plano | SSML |
|--------------|------|
| Solo contenido | Control completo de la voz. |
| Voz estándar | Estilos emocionales. |
| Sin pausas | Pausas personalizadas. |
| Pronunciación automática | Pronunciación manual. |
| Sin prosodia | Control de tono y velocidad. |

## Idea clave
- **SSML** permite controlar cómo habla el modelo, no solo qué dice.
- Se basa en **XML**.
- Permite personalizar **voz, estilo, pronunciación, pausas, prosodia y formato de lectura**.
- Se envía mediante **`SpeakSsmlAsync()`** del **SpeechSynthesizer**.
