# Use the Text to Speech API

## ¿Qué es?
API de **Azure Speech** que convierte **texto** en **voz** (*Text-to-Speech*).

## Componentes principales

### SpeechConfig
Configura la conexión con el recurso Foundry.

Contiene:
- Endpoint.
- Subscription Key.

### AudioOutputConfig
Define el destino del audio generado.

Opciones:
- 🔊 Altavoces del sistema (por defecto).
- 📁 Archivo de audio.
- 🌊 Stream de audio (si no se especifica salida).

### SpeechSynthesizer
Cliente que consume la **Text to Speech API**.

Responsabilidad:
- Convertir texto en audio.

## Flujo de funcionamiento

```text
SpeechConfig
      │
      ▼
AudioOutputConfig
      │
      ▼
SpeechSynthesizer
      │
      ▼
SpeakTextAsync()
      │
      ▼
Text to Speech API
      │
      ▼
SpeechSynthesisResult
```

## Método principal

### `SpeakTextAsync()`

- Convierte texto en voz.
- Devuelve un **SpeechSynthesisResult**.

## SpeechSynthesisResult

Propiedades importantes:

- `AudioData` → audio generado.
- `Reason` → resultado de la operación.
- `Properties`
- `ResultId`

## Valores de `Reason`

| Valor | Significado |
|--------|-------------|
| `SynthesizingAudioCompleted` | ✅ Audio generado correctamente. |
| `Canceled` | ❌ Error durante la síntesis (consultar `CancellationDetails`). |

## Flujo de uso

1. Crear `SpeechConfig`.
2. Crear `AudioOutputConfig`.
3. Crear `SpeechSynthesizer`.
4. Ejecutar `SpeakTextAsync()`.
5. Revisar `Reason`.
6. Reproducir o guardar el audio generado.

## Ejemplo

```text
Texto
   │
   ▼
SpeechSynthesizer
   │
   ▼
SpeakTextAsync()
   │
   ▼
SpeechSynthesisResult
   │
   ├── Reason = SynthesizingAudioCompleted
   └── AudioData = Audio generado
```

## Idea clave
- **SpeechConfig** → conexión al servicio.
- **AudioOutputConfig** → destino del audio.
- **SpeechSynthesizer** → genera la voz.
- **SpeakTextAsync()** → convierte texto en audio.
- El resultado se obtiene desde **SpeechSynthesisResult**, principalmente mediante `AudioData` y `Reason`.
