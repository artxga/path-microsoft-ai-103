# Comparar y combinar estrategias para optimizar modelos

No existe una única estrategia que funcione para todos los escenarios. Microsoft recomienda elegir la técnica según el problema que se desea resolver e, incluso, **combinar varias estrategias** para obtener mejores resultados. ([learn.microsoft.com](https://learn.microsoft.com/en-us/training/modules/optimize-generative-ai-model-performance/5-compare-combine-strategies?pivots=text))

---

# Comparación de estrategias

| Estrategia | ¿Qué hace? | Mejor para | ¿Requiere entrenamiento? |
|------------|------------|------------|--------------------------|
| **Prompt Engineering** | Mejora las instrucciones enviadas al modelo. | Optimizar respuestas sin modificar el modelo. | ❌ No |
| **RAG** | Recupera información externa antes de responder. | Datos privados o actualizados. | ❌ No |
| **Fine-Tuning** | Ajusta el modelo con ejemplos propios. | Adaptar comportamiento, estilo o formato. | ✅ Sí |

---

# ¿Cuándo usar cada estrategia?

## Prompt Engineering

**Úsalo cuando:**

- Necesites mejorar la calidad de las respuestas.
- Quieras cambiar el formato de salida.
- Desees definir un rol o persona.
- Busques una solución rápida y de bajo costo.

**Ventajas**

- Fácil de implementar.
- No requiere entrenamiento.
- Bajo costo.

**Limitación**

- No incorpora conocimiento nuevo al modelo.

---

## Retrieval-Augmented Generation (RAG)

**Úsalo cuando:**

- Necesites responder con información actualizada.
- Debas consultar documentos privados.
- Quieras reducir alucinaciones.
- El contenido cambie con frecuencia.

**Ventajas**

- Acceso a información externa.
- Base de conocimiento fácil de actualizar.
- Respuestas fundamentadas.

**Limitación**

- Depende de la calidad de la búsqueda y de los documentos recuperados.

---

## Fine-Tuning

**Úsalo cuando:**

- El modelo deba seguir un estilo específico.
- Necesites respuestas muy consistentes.
- Exista una tarea especializada y repetitiva.
- Prompt Engineering y RAG no sean suficientes.

**Ventajas**

- Personaliza el comportamiento del modelo.
- Reduce la necesidad de prompts extensos.

**Limitación**

- Requiere un dataset de entrenamiento y un proceso de ajuste.

---

# Combinar estrategias

Las estrategias **no son excluyentes**; pueden utilizarse juntas.

## Prompt Engineering + RAG

```text
Usuario
      │
      ▼
Prompt bien diseñado
      │
      ▼
Recuperación de documentos (RAG)
      │
      ▼
Modelo
      │
      ▼
Respuesta fundamentada
```

**Ideal para:**

- Chatbots empresariales.
- Bases de conocimiento.
- Asistentes de soporte.

---

## Prompt Engineering + Fine-Tuning

- El Fine-Tuning adapta el comportamiento del modelo.
- El Prompt Engineering ajusta la tarea específica de cada consulta.

**Ideal para:**

- Aplicaciones con un estilo de comunicación definido.
- Generación de contenido consistente.

---

## Prompt Engineering + RAG + Fine-Tuning

Es la estrategia más completa.

```text
Usuario
      │
      ▼
Prompt optimizado
      │
      ▼
RAG recupera información
      │
      ▼
Modelo Fine-Tuned
      │
      ▼
Respuesta final
```

Permite combinar:

- Contexto actualizado.
- Conocimiento privado.
- Comportamiento personalizado.
- Respuestas de alta calidad.

---

# Recomendación de Microsoft

Microsoft propone un enfoque incremental para optimizar modelos:

1. **Prompt Engineering** → mejorar las instrucciones.
2. **RAG** → agregar información externa si es necesario (mejor precisión).
3. **Fine-Tuning** → ajustar el modelo solo cuando las dos estrategias anteriores no sean suficientes (mejor consistencia).

Este orden suele ofrecer el mejor equilibrio entre costo, complejidad y rendimiento.

---

# Resumen

- No existe una estrategia universal; depende del caso de uso.
- **Prompt Engineering** optimiza las instrucciones.
- **RAG** incorpora información externa y actualizada.
- **Fine-Tuning** modifica el comportamiento del modelo mediante entrenamiento adicional.
- Las tres estrategias pueden combinarse para obtener aplicaciones de IA más precisas, útiles y personalizadas.
