# Function Calling (Function Tool)

## ¿Qué es?

**Function Calling** es una herramienta que permite a un modelo **invocar funciones definidas por el desarrollador** para interactuar con sistemas externos.

El modelo **no ejecuta la función directamente**; únicamente decide cuándo debe llamarse y con qué parámetros. La aplicación es responsable de ejecutar la función y devolver el resultado al modelo. :contentReference[oaicite:0]{index=0}

---

# ¿Por qué usar Function Calling?

Los modelos no pueden:

- Consultar bases de datos privadas.
- Consumir APIs internas.
- Modificar sistemas empresariales.
- Ejecutar acciones del mundo real.

Con **Function Calling**, el modelo puede utilizar estas capacidades mediante funciones proporcionadas por la aplicación. :contentReference[oaicite:1]{index=1}

---

# ¿Cómo funciona?

```text
Usuario
    │
    ▼
 Prompt
    │
    ▼
 Modelo
    │
¿Necesita una función?
    │
 ┌──┴──────────────┐
 │ Sí             │ No
 ▼                ▼
Solicitud de      Respuesta
Function Call
 │
 ▼
Aplicación ejecuta
la función
 │
 ▼
Resultado
 │
 ▼
Modelo genera
la respuesta final
```

Flujo:

1. El usuario envía un prompt.
2. El modelo detecta que necesita información o una acción externa.
3. Devuelve el **nombre de la función** y los **argumentos**.
4. La aplicación ejecuta la función.
5. La aplicación devuelve el resultado al modelo.
6. El modelo genera la respuesta final. :contentReference[oaicite:2]{index=2}

---

# Componentes de una función

Cada función debe definir:

- **Name:** nombre de la función.
- **Description:** explica cuándo debe utilizarse.
- **Parameters:** parámetros que recibe (JSON Schema).
- **Required:** parámetros obligatorios.

Ejemplo conceptual:

```text
Nombre:
getWeather

Descripción:
Obtiene el clima actual de una ciudad.

Parámetros:
- location (string)
- unit (string, opcional)
```

El modelo utiliza esta información para decidir si debe llamar a la función. :contentReference[oaicite:3]{index=3}

# ¿Qué puede hacer?

Permite integrar el modelo con:

- APIs REST.
- Bases de datos.
- ERP o CRM.
- Sistemas internos.
- Servicios externos.
- Aplicaciones empresariales.

---

# Casos de uso

- Consultar inventario.
- Obtener el clima.
- Buscar información en una base de datos.
- Reservar hoteles o vuelos.
- Crear pedidos.
- Enviar correos electrónicos.
- Consultar información de clientes.
- Ejecutar procesos de negocio.

---

# Ejemplo

**Prompt**

> ¿Hay stock del producto X?

**Proceso**

1. El modelo detecta que necesita consultar el inventario.
2. Solicita ejecutar `getInventory(producto="X")`.
3. La aplicación consulta la base de datos.
4. Devuelve el resultado.
5. El modelo responde:

> Sí, hay 25 unidades disponibles.

---

# Buenas prácticas

- Escribir descripciones claras para cada función.
- Definir correctamente los parámetros requeridos.
- Validar los argumentos recibidos.
- No devolver información sensible innecesaria.
- Manejar errores de ejecución y devolver mensajes útiles. :contentReference[oaicite:4]{index=4}

---

# Consideraciones

- El modelo **no ejecuta** las funciones.
- La lógica de negocio permanece en la aplicación.
- La aplicación controla permisos, seguridad y acceso a datos.
- Una solicitud puede requerir **varias llamadas a funciones** antes de generar la respuesta final. :contentReference[oaicite:5]{index=5}

---

# Resumen

- **Function Calling** permite conectar un modelo con funciones creadas por el desarrollador.
- El modelo decide **cuándo** llamar una función y **qué parámetros** enviar.
- La aplicación ejecuta la función y devuelve el resultado al modelo.
- Es la herramienta ideal para integrar IA con APIs, bases de datos y sistemas empresariales.
- El modelo nunca ejecuta código de negocio por sí solo; solo coordina el flujo de llamadas.
