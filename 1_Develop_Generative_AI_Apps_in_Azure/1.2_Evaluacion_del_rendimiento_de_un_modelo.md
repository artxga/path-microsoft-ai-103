# Evaluación del rendimiento de un modelo

## ¿Por qué evaluar un modelo?

La evaluación permite determinar si un modelo responde correctamente a las necesidades del caso de uso antes de ponerlo en producción.

Puede realizarse de dos formas:

- **Evaluación manual**
- **Evaluación automatizada**

---

# Evaluación manual

Consiste en probar el modelo directamente en el **Playground**.

### Proceso

1. Escribir un prompt.
2. Analizar la respuesta generada.
3. Ajustar el prompt o el modelo si es necesario.
4. Repetir el proceso hasta obtener resultados satisfactorios.

### ¿Qué evaluar?

- Precisión de la respuesta.
- Relevancia.
- Coherencia.
- Cumplimiento de instrucciones.
- Calidad general.

> Es útil durante las primeras etapas del desarrollo.

---

# Evaluación automatizada

Permite evaluar el modelo usando un conjunto de pruebas de manera consistente y repetible.

### Ventajas

- Reduce el trabajo manual.
- Produce resultados objetivos.
- Facilita comparar distintos modelos o versiones.
- Permite medir el rendimiento a gran escala.

---

# Dataset de evaluación

La evaluación automática requiere un conjunto de datos con ejemplos.

Cada registro suele contener:

- **Prompt (entrada)**
- **Respuesta esperada (opcional)**
- **Respuesta generada por el modelo**

Estos datos sirven para calcular métricas de calidad.

---

# Métricas de evaluación

Microsoft Foundry ofrece métricas basadas en IA para distintos aspectos del rendimiento.

## Calidad

Evalúa si la respuesta:

- Es correcta.
- Es relevante.
- Sigue las instrucciones.
- Resuelve adecuadamente la solicitud.

---

## Groundedness

Mide si la respuesta está respaldada por la información proporcionada.

- Detecta respuestas inventadas (*hallucinations*).
- Es especialmente importante en aplicaciones con **RAG (Retrieval-Augmented Generation)**.

---

## Similitud

Compara la respuesta generada con una respuesta de referencia.

Permite medir qué tan parecidas son en significado, aunque no utilicen exactamente las mismas palabras.

---

## Fluidez

Evalúa:

- Gramática.
- Claridad.
- Naturalidad.
- Legibilidad.

---

## Coherencia

Determina si la respuesta:

- Tiene sentido.
- Mantiene una estructura lógica.
- No presenta contradicciones.

---

# Comparación de modelos

Las evaluaciones permiten comparar distintos modelos utilizando las mismas pruebas.

Se pueden identificar diferencias en:

- Calidad.
- Seguridad.
- Coste.
- Rendimiento.

Esto facilita seleccionar el modelo más adecuado para un escenario específico.

---

# Ciclo de mejora

La evaluación es un proceso iterativo:

1. Probar el modelo.
2. Medir resultados.
3. Identificar problemas.
4. Ajustar prompts o cambiar de modelo.
5. Volver a evaluar.

El objetivo es mejorar continuamente el desempeño de la aplicación.

---

# Resumen

- La evaluación puede ser **manual** o **automatizada**.
- La evaluación manual se realiza desde el Playground.
- La evaluación automática utiliza datasets y métricas.
- Las métricas analizan aspectos como **calidad, groundedness, similitud, fluidez y coherencia**.
- Evaluar periódicamente ayuda a seleccionar y optimizar el mejor modelo para cada caso de uso.
