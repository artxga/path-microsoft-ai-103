# Fine-Tuning

## ¿Qué es?

**Fine-Tuning** es el proceso de **reentrenar un modelo preentrenado con un conjunto de datos específico** para adaptarlo a una tarea, dominio o estilo determinado.

En lugar de entrenar un modelo desde cero, se parte de un modelo base y se ajusta con ejemplos propios. ([learn.microsoft.com](https://learn.microsoft.com/en-us/training/modules/optimize-generative-ai-model-performance/4-fine-tune-model?pivots=text))

---

# ¿Por qué usar Fine-Tuning?

Aunque el **Prompt Engineering** suele ser la primera opción para mejorar un modelo, existen casos en los que no es suficiente.

El Fine-Tuning permite:

- Adaptar el modelo a un dominio específico.
- Enseñar formatos de respuesta consistentes.
- Mejorar tareas repetitivas.
- Reducir la necesidad de prompts largos y complejos.
- Personalizar el comportamiento del modelo.

---

# ¿Cómo funciona?

```text
Modelo base
      │
      ▼
Dataset de entrenamiento
(prompts + respuestas esperadas)
      │
      ▼
Proceso de Fine-Tuning
      │
      ▼
Modelo ajustado
(Fine-Tuned Model)
```

### Flujo

1. Seleccionar un modelo base.
2. Preparar un dataset de entrenamiento.
3. Ejecutar el proceso de Fine-Tuning.
4. Validar el modelo ajustado.
5. Desplegar el nuevo modelo.

---

# Dataset de entrenamiento

El dataset contiene ejemplos de cómo debe responder el modelo.

Generalmente incluye pares de:

- **Prompt**
- **Respuesta esperada**

**Ejemplo**

```text
Prompt:
Resume este texto para un director ejecutivo.

Respuesta:
...
```

Mientras mayor sea la calidad y diversidad de los ejemplos, mejores serán los resultados.

---

# ¿Cuándo usar Fine-Tuning?

Es recomendable cuando se necesita:

- Un estilo de respuesta específico.
- Un formato consistente.
- Mejor desempeño en una tarea concreta.
- Adaptar el modelo a terminología especializada.
- Reducir la cantidad de instrucciones en cada prompt.

---

# ¿Cuándo NO usar Fine-Tuning?

No es la mejor opción cuando el problema consiste en acceder a información nueva o privada.

En esos casos es preferible utilizar:

- **Prompt Engineering**, si basta con mejorar las instrucciones.
- **RAG (Retrieval-Augmented Generation)**, si el conocimiento cambia con frecuencia o proviene de documentos externos.

---

# Fine-Tuning vs RAG

| Fine-Tuning | RAG |
|-------------|-----|
| Modifica el comportamiento del modelo. | Agrega contexto antes de responder. |
| Requiere un dataset de entrenamiento. | Requiere una base de conocimiento. |
| Ideal para cambiar estilo, formato o comportamiento. | Ideal para responder con información actualizada o privada. |
| Los cambios permanecen en el modelo ajustado. | El modelo sigue siendo el mismo; cambia el contexto recibido. |

---

# Buenas prácticas

- Utilizar datos de entrenamiento de alta calidad.
- Eliminar ejemplos incorrectos o inconsistentes.
- Incluir suficientes ejemplos representativos.
- Evaluar el modelo antes y después del Fine-Tuning.
- Aplicar Fine-Tuning solo cuando Prompt Engineering o RAG no sean suficientes.

---

# Recomendación de Microsoft

Antes de realizar Fine-Tuning, Microsoft recomienda intentar mejorar el rendimiento mediante:

1. **Prompt Engineering**.
2. **RAG (Retrieval-Augmented Generation)**.
3. **Fine-Tuning**, solo si las opciones anteriores no satisfacen los requisitos.

---

# Resumen

- **Fine-Tuning** consiste en reentrenar un modelo base con ejemplos propios.
- Permite adaptar el modelo a un dominio, formato o comportamiento específico.
- Requiere un dataset con **prompts y respuestas esperadas**.
- No es la mejor opción para incorporar información actualizada; para ello es preferible utilizar **RAG**.
- Microsoft recomienda recurrir al Fine-Tuning solo después de optimizar el modelo con Prompt Engineering y RAG.
