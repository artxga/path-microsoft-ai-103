# Tipos de despliegue de modelos

En Microsoft AI Foundry existen **dos niveles de clasificación** para los despliegues:

1. **Modo de despliegue** (quién administra la infraestructura).
2. **Tipo de despliegue** (facturación, residencia de datos y rendimiento).

---

# 1. Modos de despliegue

## Serverless API (Recomendado)

**¿Qué es?**
- Microsoft hospeda y administra toda la infraestructura.
- Solo consumes el modelo mediante una API.
- No necesitas configurar máquinas virtuales ni GPUs.

### Ventajas
- ✅ Despliegue rápido.
- ✅ Sin administrar infraestructura.
- ✅ Escalado automático.
- ✅ Pago por uso (tokens o solicitudes).

### Ideal para
- Prototipos.
- Desarrollo.
- Aplicaciones con demanda variable.
- La mayoría de casos de uso.

---

## Managed Compute (Managed Online Endpoint)

**¿Qué es?**
- El modelo se ejecuta en recursos de Azure administrados por el usuario.
- Debes elegir el hardware y la capacidad de cómputo.

### Configuración
- Tipo de máquina (CPU/GPU).
- Número de instancias.
- Escalado.
- Recursos de cómputo.

### Ventajas
- Mayor control del rendimiento.
- Personalización de recursos.
- Adecuado para producción y cargas elevadas.

### Desventajas
- Mayor complejidad.
- Mayor costo.

---

# Comparación de modos

| Característica | Serverless API | Managed Compute |
|----------------|----------------|-----------------|
| Infraestructura | Microsoft | Usuario |
| Configuración | Baja | Alta |
| Escalado | Automático | Configurable |
| Pago | Por uso | Recursos aprovisionados |
| Ideal para | Desarrollo y apps generales | Producción y cargas intensivas |

---

# 2. Tipos de despliegue

Los modelos pueden ofrecer distintos tipos de despliegue según sus necesidades de **residencia de datos**, **facturación** y **rendimiento**.

| Tipo | Facturación | Residencia de datos | Uso principal |
|------|-------------|---------------------|---------------|
| **Global Standard** | Pago por token | Cualquier región | Uso general |
| **Global Provisioned** | PTUs reservadas | Cualquier región | Alto rendimiento |
| **Global Batch** | Batch (50% descuento) | Cualquier región | Procesamiento masivo |
| **Data Zone Standard** | Pago por token | Zona de datos (UE/EE. UU.) | Cumplimiento normativo |
| **Data Zone Provisioned** | PTUs reservadas | Zona de datos | Alto rendimiento + cumplimiento |
| **Data Zone Batch** | Batch | Zona de datos | Procesamiento masivo con cumplimiento |
| **Standard (Regional)** | Pago por token | Una región específica | Residencia regional |
| **Regional Provisioned** | PTUs reservadas | Una región específica | Alto rendimiento regional |
| **Developer** | Pago por token | Cualquier región | Evaluación de modelos fine-tuned |

---

## Global Standard ⭐ (Recomendado)

- Pago por token.
- Puede utilizar cualquier región de Azure.
- Escalado automático.
- Mayor cuota disponible.

**Ideal para:** la mayoría de aplicaciones.

---

## Global Provisioned

- Reserva capacidad mediante **PTUs (Provisioned Throughput Units)**.
- Rendimiento y latencia predecibles.

**Ideal para:** aplicaciones críticas con alto tráfico.

---

## Global Batch

- Procesamiento asíncrono por lotes.
- Trabajos completados en un plazo de hasta 24 horas.
- Aproximadamente **50% más económico**.

**Ideal para:** grandes volúmenes de procesamiento sin necesidad de respuesta inmediata.

---

## Data Zone Standard

- Pago por token.
- Garantiza que los datos permanezcan dentro de una **zona de datos** (por ejemplo, UE o EE. UU.).

**Ideal para:** cumplimiento normativo.

---

## Data Zone Provisioned

- Igual que Data Zone Standard, pero con **PTUs reservadas**.

**Ideal para:** alto rendimiento con requisitos regulatorios.

---

## Data Zone Batch

- Procesamiento por lotes dentro de una zona de datos.

**Ideal para:** grandes trabajos asíncronos con residencia de datos.

---

## Standard (Regional)

- Despliegue en una única región.
- Pago por token.

**Ideal para:** requisitos de residencia regional o aplicaciones de bajo volumen.

---

## Regional Provisioned

- Despliegue regional con **PTUs reservadas**.

**Ideal para:** aplicaciones regionales de alto rendimiento.

---

## Developer

- Pago por token.
- Solo para evaluar modelos **fine-tuned**.
- No está pensado para producción.

---

# ¿Qué son las PTUs?

**Provisioned Throughput Units (PTUs)** son unidades de capacidad reservada.

En lugar de pagar por cada token, reservas capacidad para obtener:

- Rendimiento constante.
- Baja latencia.
- Throughput garantizado.

---

# Recomendación de Microsoft

> Siempre que sea posible, utiliza **Global Standard**, ya que ofrece la mayor compatibilidad, escalado automático y la cuota más alta. Utiliza otros tipos de despliegue solo cuando necesites residencia de datos, capacidad garantizada o procesamiento por lotes.

---

# Resumen

- Existen **dos modos de despliegue**:
  - **Serverless API:** Microsoft administra la infraestructura.
  - **Managed Compute:** el usuario administra los recursos de Azure.

- Dentro de esos modos, los modelos pueden ofrecer distintos **tipos de despliegue**:
  - **Standard:** pago por token.
  - **Provisioned:** capacidad reservada mediante PTUs.
  - **Batch:** procesamiento masivo asíncrono.
  - **Global:** cualquier región.
  - **Data Zone:** residencia en una zona de datos (UE/EE. UU.).
  - **Regional:** residencia en una región específica.
  - **Developer:** evaluación de modelos ajustados.
