# Mitigar daños (Mitigate Harms)

Después de **identificar** y **medir** los posibles daños de una aplicación de IA, el siguiente paso es **mitigarlos** para reducir los riesgos antes de poner el sistema en producción.

La mitigación consiste en aplicar controles técnicos y de diseño que hagan que la aplicación sea más **segura, confiable y responsable**.

---

# Estrategias de mitigación

Microsoft recomienda aplicar varias estrategias en conjunto, ya que ninguna elimina completamente todos los riesgos.

## 1. Mejorar el modelo

Elegir el modelo más adecuado para el caso de uso.

Antes de desplegarlo, compara aspectos como:

- Calidad.
- Seguridad.
- Rendimiento.
- Coste.

Un modelo con mejores métricas de seguridad puede generar menos contenido dañino.

---

## 2. Mejorar el Prompt Engineering

Modificar los prompts para guiar mejor el comportamiento del modelo.

Ejemplos:

- Definir claramente el rol del asistente.
- Especificar restricciones.
- Solicitar respuestas fundamentadas.
- Limitar determinados tipos de contenido.

**Ejemplo**

```text
Responde únicamente utilizando la información proporcionada.
Si no encuentras la respuesta, indica que no dispones de suficiente información.
```

Esto ayuda a reducir las alucinaciones.

---

## 3. Utilizar RAG (Retrieval-Augmented Generation)

En lugar de confiar únicamente en el conocimiento del modelo, proporcionar contexto recuperado desde documentos confiables.

**Beneficios**

- Reduce alucinaciones.
- Mejora la precisión.
- Permite responder con información actualizada.
- Fundamenta las respuestas.

---

## 4. Fine-Tuning

Cuando Prompt Engineering y RAG no son suficientes, puede ajustarse el modelo mediante Fine-Tuning para adaptar su comportamiento.

Es útil para:

- Respuestas consistentes.
- Estilos específicos.
- Casos de uso especializados.

---

## 5. Implementar filtros de contenido (Content Filters)

Los filtros permiten detectar y bloquear contenido potencialmente dañino.

Pueden filtrar categorías como:

- Violencia.
- Discurso de odio.
- Contenido sexual.
- Autolesiones.

El objetivo es impedir que el modelo genere o procese contenido inapropiado.

---

## 6. Evaluación continua

La mitigación no termina con el despliegue.

Se recomienda:

- Ejecutar evaluaciones periódicas.
- Analizar nuevas versiones del modelo.
- Revisar cambios en los datos.
- Supervisar el comportamiento de la aplicación.

---

# Content Filters

Microsoft AI Foundry incorpora **Content Filters** para moderar tanto:

- **Prompts del usuario** (entrada).
- **Respuestas del modelo** (salida).

```text
Usuario
    │
    ▼
Filtro de entrada
    │
    ▼
Modelo
    │
    ▼
Filtro de salida
    │
    ▼
Respuesta final
```

Los filtros ayudan a bloquear solicitudes o respuestas que puedan generar riesgos.

---

# Mitigación por capas

Microsoft recomienda aplicar varias capas de protección en lugar de depender de una única técnica.

```text
Usuario
      │
      ▼
Prompt Engineering
      │
      ▼
Content Filters
      │
      ▼
RAG
      │
      ▼
Modelo
      │
      ▼
Evaluación
      │
      ▼
Respuesta
```

Cada capa reduce un tipo diferente de riesgo.

---

# Buenas prácticas

- Elegir el modelo más seguro para el escenario.
- Diseñar prompts claros y restrictivos.
- Utilizar RAG para información privada o actualizada.
- Aplicar Fine-Tuning solo cuando sea necesario.
- Configurar filtros de contenido.
- Evaluar continuamente el sistema después del despliegue.

---

# Recomendación de Microsoft

La mitigación debe abordarse de forma **integral**, combinando distintas técnicas:

1. Seleccionar el modelo adecuado.
2. Mejorar el Prompt Engineering.
3. Incorporar RAG cuando se necesite contexto externo.
4. Aplicar Fine-Tuning si es necesario.
5. Configurar filtros de contenido.
6. Evaluar continuamente el comportamiento del sistema.

---

# Resumen

- Mitigar daños consiste en reducir los riesgos identificados durante la evaluación de una aplicación de IA.
- Microsoft recomienda combinar varias estrategias:
  - Elegir un modelo adecuado.
  - Optimizar los prompts.
  - Utilizar RAG.
  - Aplicar Fine-Tuning cuando sea necesario.
  - Configurar filtros de contenido.
  - Realizar evaluaciones continuas.
- La mitigación es un proceso continuo que debe mantenerse durante todo el ciclo de vida de la aplicación.
