# What is Azure Document Intelligence?

Ejercicio: https://microsoftlearning.github.io/mslearn-ai-information-extraction/Instructions/Exercises/03-document-intelligence.html

**Azure Document Intelligence** es un servicio de IA de **Microsoft Foundry** que utiliza **OCR** y modelos de **Deep Learning** para extraer información estructurada desde documentos.

Permite obtener automáticamente:

- Texto
- Pares clave-valor (Key-Value Pairs)
- Tablas
- Casillas de selección (Selection Marks)
- Estructura del documento
- Coordenadas (Bounding Boxes)

Todo se devuelve en formato **JSON estructurado**, preservando la organización original del documento.

---

# ¿Cómo funciona?

Document Intelligence utiliza **OCR (Optical Character Recognition)** para detectar el contenido de un documento.

Durante el proceso:

1. Detecta texto y elementos visuales.
2. Crea **Bounding Boxes** alrededor de cada elemento encontrado.
3. Registra las coordenadas de cada objeto.
4. Analiza la estructura del documento mediante modelos de IA.
5. Devuelve toda la información organizada en JSON.

---

# Bounding Boxes

Los **Bounding Boxes** representan la ubicación exacta de un elemento dentro de la página.

Ejemplo:

```text
┌─────────────────────────────┐
│                             │
│  Invoice Number             │
│  ┌──────────────────────┐   │
│  │ INV-2025-001         │   │
│  └──────────────────────┘   │
│                             │
└─────────────────────────────┘
```

Cada elemento incluye sus coordenadas dentro del documento.

Esto permite:

- Resaltar información encontrada.
- Validar resultados.
- Mantener la estructura original.
- Facilitar revisiones humanas.

# ¿Por qué usar Document Intelligence?

Entrenar un modelo propio para leer documentos requiere:

- Grandes volúmenes de datos etiquetados.
- Conocimientos de Deep Learning.
- Mucho tiempo de entrenamiento.
- Alta capacidad de cómputo.

Document Intelligence ya incluye modelos previamente entrenados sobre miles de documentos, permitiendo obtener alta precisión sin necesidad de entrenar desde cero.

# Componentes de Document Intelligence

Document Intelligence se divide en **tres grandes categorías** de modelos.

## 1. Document Analysis Models

Analizan la estructura general del documento.

Extraen:

- Texto
- Idioma
- Layout
- Tablas
- Selection Marks

Los principales modelos son:

- **Read Model**
  - OCR
  - Detección de idioma
  - Extracción de texto

- **Layout Model**
  - Todo lo anterior
  - Tablas
  - Encabezados
  - Párrafos
  - Estructura del documento

---

## 2. Prebuilt Models

Modelos ya entrenados para documentos comunes.

Ejemplos:

- Facturas
- Recibos
- Documentos de identidad
- Formularios de impuestos
- Contratos
- Tarjetas de presentación
- Entre otros

No requieren entrenamiento.

---

## 3. Custom Models

Permiten entrenar modelos propios para documentos específicos de una empresa.

Tipos:

### Custom Template Model

- Muy rápido.
- Económico.
- Ideal para documentos con diseño fijo.

Ejemplo:

Siempre la misma factura con exactamente el mismo formato.

---

### Custom Neural Model

Utiliza modelos neuronales.

Ventajas:

- Mayor precisión.
- Tolera variaciones en el diseño.

Ideal para documentos similares pero no idénticos.

---

### Composed Models

Permiten combinar varios modelos personalizados.

Ejemplo:

```
Factura A
Factura B
Factura C
      │
      ▼
Composed Model
```

El sistema detecta automáticamente cuál utilizar.

---

### Custom Classifier

Clasifica automáticamente el tipo de documento antes de analizarlo.

Ejemplo:

```
Documento
      │
      ▼
¿Factura?
¿Recibo?
¿Contrato?
¿Pasaporte?
```

Después envía el documento al modelo correspondiente.

# Recursos de Azure

Existen dos opciones.

## Foundry Resource

Un único recurso para múltiples servicios.

Incluye:

- Document Intelligence
- Speech
- Translator
- Vision
- OpenAI
- Content Understanding
- otros servicios de Foundry

Todos utilizan:

- mismo endpoint
- misma clave

Ideal para proyectos que usan varios servicios.

---

## Document Intelligence Resource

Servicio dedicado únicamente a Document Intelligence.

Se utiliza cuando no se necesitan otros servicios de IA.

# Flujo de procesamiento

```text
Documento
      │
      ▼
OCR
      │
      ▼
Bounding Boxes
      │
      ▼
Modelos de IA
      │
      ├── Texto
      ├── Tablas
      ├── Key-Value
      ├── Layout
      ├── Selection Marks
      └── Idioma
      │
      ▼
JSON estructurado
```

# Ideas clave

- **Document Intelligence** combina **OCR + IA** para convertir documentos en datos estructurados.
- Devuelve resultados en **JSON**, incluyendo texto, tablas, pares clave-valor y coordenadas (**Bounding Boxes**).
- Incluye modelos **Document Analysis**, **Prebuilt** y **Custom**.
- Puede utilizarse mediante **REST API**, **SDK**, **Document Intelligence Studio** o **Microsoft Foundry**.
- Admite documentos e imágenes en formatos comunes como **PDF, JPEG, PNG, BMP y TIFF**, respetando límites de tamaño y resolución.
