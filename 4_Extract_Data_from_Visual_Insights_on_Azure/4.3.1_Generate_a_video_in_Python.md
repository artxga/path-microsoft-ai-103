# Generate video in Python (Sora 2)

La generación de video con **Sora 2** desde Python utiliza el **OpenAI Python SDK**.

La generación es **asíncrona**, por lo que el flujo consiste en:

1. Crear el trabajo (job).
2. Consultar periódicamente su estado (polling).
3. Descargar el video cuando finalice.

---

# Flujo general

```
Prompt
      │
      ▼
client.videos.create()
      │
      ▼
Video Job
      │
      ▼
Polling (retrieve)
      │
      ▼
completed
      │
      ▼
download_content()
      │
      ▼
output.mp4
```

---

# Paso 1. Crear el trabajo

```python
import time

video = client.videos.create(
    model="sora-2",
    prompt="A robot walks through a rainy city street at dusk, neon signs reflecting in puddles",
    size="1280x720",
    seconds="4",
)

print(f"Video creation started. ID: {video.id}")
```

## Parámetros

| Parámetro | Descripción |
|-----------|-------------|
| `model` | Modelo (`sora-2` o `sora-2-pro`) |
| `prompt` | Descripción del video |
| `size` | Resolución |
| `seconds` | Duración |
| `input_reference` | Imagen de referencia (opcional) |

---

# Paso 2. Consultar el estado (Polling)

Como la generación es asíncrona, hay que consultar el estado periódicamente.

```python
while video.status not in ["completed", "failed", "cancelled"]:
    print(f"Status: {video.status}. Waiting...")
    time.sleep(20)
    video = client.videos.retrieve(video.id)
```

Se consulta el estado mediante:

```python
client.videos.retrieve(video.id)
```

---

# Paso 3. Descargar el video

Cuando el estado sea **completed**, puede descargarse.

```python
if video.status == "completed":
    content = client.videos.download_content(
        video.id,
        variant="video"
    )

    content.write_to_file("output.mp4")

    print("Video saved to output.mp4")
```

---

# Ejemplo completo

```python
import time

video = client.videos.create(
    model="sora-2",
    prompt="A robot walks through a rainy city street at dusk, neon signs reflecting in puddles",
    size="1280x720",
    seconds="4",
)

print(video.id)

while video.status not in ["completed", "failed", "cancelled"]:
    print(video.status)
    time.sleep(20)
    video = client.videos.retrieve(video.id)

if video.status == "completed":
    content = client.videos.download_content(
        video.id,
        variant="video"
    )

    content.write_to_file("output.mp4")
```

# Generar un video desde una imagen

Puede utilizarse una imagen como primer fotograma mediante:

```
input_reference
```

Ejemplo:

```python
video = client.videos.create(
    model="sora-2",
    prompt="The camera slowly pans across the landscape as clouds drift overhead",
    size="1280x720",
    seconds="4",
    input_reference=open("landscape.png", "rb"),
)
```

## Requisitos

La imagen debe:

- Tener exactamente la misma resolución del video.
- Ser JPEG, PNG o formato compatible.

Actualmente **no se permiten imágenes con rostros humanos**.

Sí pueden utilizarse:

- Paisajes
- Objetos
- Personajes animados

---

# Remix de un video

Es posible modificar un video ya generado manteniendo gran parte de su estructura.

```python
video = client.videos.remix(
    video_id="video_abc123",
    prompt="Change the color palette to warm sunset tones",
)
```

Solo es necesario indicar:

- El ID del video original.
- El nuevo prompt describiendo el cambio.

---

# Estados de un trabajo

| Estado | Significado |
|----------|------------|
| `queued` | Esperando para comenzar |
| `in_progress` | Generándose |
| `completed` | Finalizado correctamente |
| `failed` | Error durante la generación |
| `cancelled` | Cancelado |

---

# Manejo de errores

Si el trabajo falla:

```python
video.error
```

contiene información sobre la causa del error.

---

# Consideraciones importantes

## Límite de trabajos

Máximo:

**2 trabajos simultáneos**

---

## Expiración

Los videos permanecen disponibles para descarga durante:

**24 horas**

---

## Resolución

Las imágenes utilizadas como referencia deben tener exactamente la misma resolución del video generado.

Ejemplo:

Video:

```
1280x720
```

Imagen:

```
1280x720
```

---

## Moderación

Todos los prompts pasan por filtros de contenido.

Si el contenido es considerado dañino o prohibido:

- no se genera el video.

# Ideas clave para el examen

- La generación de video con **Sora 2** es **asíncrona**.
- Flujo:
  1. `client.videos.create()`
  2. `client.videos.retrieve()`
  3. `client.videos.download_content()`
- Se debe realizar **polling** hasta que el estado sea `completed`.
- Estados posibles:
  - `queued`
  - `in_progress`
  - `completed`
  - `failed`
  - `cancelled`
- `input_reference` permite utilizar una imagen como primer fotograma.
- `client.videos.remix()` modifica un video existente usando su `video_id`.
- Máximo **2 trabajos simultáneos**.
- Los videos generados permanecen disponibles durante **24 horas**.
