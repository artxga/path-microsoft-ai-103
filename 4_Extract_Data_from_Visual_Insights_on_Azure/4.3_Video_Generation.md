# Generate video from a prompt (Sora 2)

Una vez desplegado un modelo **Sora 2** en Microsoft Foundry, puedes generar videos a partir de descripciones en lenguaje natural.

La generación de video es un proceso **asíncrono**:

1. Envías una solicitud con un prompt y parámetros.
2. El servicio inicia un trabajo (job).
3. Esperas a que termine.
4. Descargas el video generado.

---

# Flujo general

```
Prompt
      │
      ▼
Solicitud al modelo Sora 2
      │
      ▼
Job de generación (asíncrono)
      │
      ▼
Esperar finalización
      │
      ▼
Video generado
```

---

# Parámetros de generación

| Parámetro | Descripción | Valores |
|-----------|-------------|----------|
| `prompt` | Descripción del video | Texto (obligatorio) |
| `model` | Modelo utilizado | `sora-2`, `sora-2-pro` |
| `size` | Resolución del video | `1280x720`, `720x1280` |
| `seconds` | Duración | `4`, `8`, `12` segundos |
| `input_reference` | Imagen de referencia para el primer frame | JPEG, PNG, WebP |
| `remix_video_id` | ID de un video previamente generado para modificarlo | Video ID |

---

# Modelos disponibles

- `sora-2`
- `sora-2-pro`

---

# Resoluciones soportadas

## Horizontal

```
1280 x 720
```

## Vertical

```
720 x 1280
```

---

# Duraciones soportadas

- 4 segundos (por defecto)
- 8 segundos
- 12 segundos

> **Recomendación:** para obtener mejores resultados, suele ser preferible generar **dos clips de 4 segundos** y unirlos, en lugar de generar directamente un clip de 8 segundos.

---

# Generar un video desde el Playground

En Microsoft Foundry Portal:

1. Abrir el modelo **Sora 2** desplegado.
2. Ir al **Playground**.
3. Escribir el prompt.
4. Configurar:
   - Resolución
   - Duración
5. Seleccionar **Generate**.
6. Esperar entre **1 y 5 minutos** aproximadamente.
7. Descargar el video generado.

---

# Moderación de contenido

Las APIs de generación incluyen filtros de contenido.

Si el prompt contiene contenido considerado dañino o prohibido, **no se generará el video**.

---

# Cómo escribir buenos prompts

La idea es describir la escena como si estuvieras dándole instrucciones a un director de fotografía.

Un buen prompt suele incluir:

- Cámara
- Sujeto
- Acción
- Iluminación
- Estilo

---

## 1. Cámara

Describe el tipo de plano.

Ejemplos:

- Wide shot
- Medium shot
- Close-up
- Low angle
- High angle

---

## 2. Sujeto

Describe claramente el protagonista.

Ejemplo:

```
An old Swedish man
```

En lugar de simplemente:

```
A man
```

---

## 3. Acción

Describe exactamente qué ocurre.

Mejor:

```
Cyclist pedals three times,
brakes,
stops at the crosswalk.
```

Peor:

```
Cyclist moves quickly.
```

---

## 4. Iluminación y colores

Especifica el ambiente.

Ejemplos:

- Neon lights
- Warm sunset
- Volumetric lighting
- Soft morning light
- Wet asphalt reflections

---

## 5. Estilo

Define el estilo desde el inicio.

Ejemplos:

- 90s documentary
- 1970s film
- Handheld documentary
- Cinematic
- Anime
- Noir

---

# Ejemplo de buen prompt

```text
In a 90s documentary-style interview,
an old Swedish man sits in a study
and says,

"I still remember when I was young."
```

¿Por qué funciona?

- Define el estilo.
- Describe el sujeto.
- Describe el escenario.
- Incluye el diálogo.

---

# Prompts débiles vs fuertes

| Débil | Fuerte |
|--------|---------|
| A beautiful street at night | Wet asphalt, zebra crosswalk, neon signs reflecting in puddles |
| Person moves quickly | Cyclist pedals three times, brakes, and stops |
| Cinematic look | Anamorphic 2.0x lens, shallow DOF, volumetric light |

---

# Imágenes de referencia

Puede proporcionarse una imagen para controlar el primer fotograma del video.

Parámetro:

```
input_reference
```

Formatos soportados:

- JPEG
- PNG
- WebP

La resolución debe coincidir con la del video:

- 1280×720
- 720×1280

El modelo utilizará la imagen como punto de partida y continuará la escena siguiendo el prompt.

---

# Remix de videos

Sora permite modificar un video previamente generado sin volver a crearlo desde cero.

Se utiliza:

```
remix_video_id
```

Proceso:

1. Generar un video.
2. Obtener su Video ID.
3. Enviar una nueva solicitud indicando:
   - `remix_video_id`
   - Nuevo prompt

---

## Buenas prácticas para Remix

Realizar un único cambio específico.

Ejemplos:

- Same shot, switch to 85mm lens.
- Same lighting, new palette: teal, sand, rust.

Cuanto más pequeño sea el cambio, mayor será la fidelidad al video original.

---

# Buenas prácticas

- Un solo movimiento de cámara por toma.
- Una acción principal por escena.
- Dividir acciones largas en pequeños pasos.
- Reutilizar la descripción de personajes para mantener continuidad.
- Ajustar ligeramente los prompts entre generaciones para mejorar el resultado.
- Experimentar: distintas ejecuciones pueden producir variaciones creativas.

---

# Flujo completo

```
Escribir prompt
        │
        ▼
Elegir modelo
(sora-2 / sora-2-pro)
        │
        ▼
Configurar
• Resolución
• Duración
• Imagen de referencia (opcional)
• Remix (opcional)
        │
        ▼
Enviar solicitud
        │
        ▼
Job asíncrono
        │
        ▼
Esperar 1–5 minutos
        │
        ▼
Video generado
```

- La generación de video con **Sora 2** es **asíncrona**.
- Modelos disponibles:
  - `sora-2`
