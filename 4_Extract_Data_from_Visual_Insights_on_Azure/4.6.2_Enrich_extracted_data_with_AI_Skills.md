# Enrich Extracted Data with AI Skills (Azure AI Search)

Una de las características más poderosas de **Azure AI Search** es el **Enrichment Pipeline**, donde un **Indexer** ejecuta un conjunto de **AI Skills (Skillset)** para enriquecer automáticamente los documentos antes de almacenarlos en el índice.

Cada Skill toma información del documento, la procesa y agrega nuevos campos que podrán utilizarse posteriormente para búsquedas, filtros o aplicaciones de IA como RAG.

---

# Arquitectura

```
Data Source
      │
      ▼
Indexer
      │
      ▼
Document Cracking
      │
      ▼
Skillset
      │
      ├── Skill 1
      ├── Skill 2
      ├── Skill 3
      └── ...
      │
      ▼
Documento enriquecido
      │
      ▼
Search Index
```

---

# ¿Qué es un Skillset?

Un **Skillset** es un conjunto ordenado de **AI Skills**.

El Indexer ejecuta cada Skill en secuencia.

Cada Skill:

- recibe datos del documento
- genera nueva información
- agrega nuevos campos al documento
- deja esos resultados disponibles para la siguiente Skill

Es decir, el documento se va enriqueciendo paso a paso.

---

# Flujo de enriquecimiento

```
Documento

↓

Skill 1

↓

Documento enriquecido

↓

Skill 2

↓

Documento más enriquecido

↓

Skill 3

↓

Documento final
```

Cada Skill puede utilizar la salida producida por las anteriores.

---

# Built-in Skills

Azure AI Search incluye numerosas **Built-in Skills**, listas para usar.

Estas Skills aprovechan servicios de **Azure AI Foundry Tools**, como:

- Azure AI Vision
- Azure AI Language

No es necesario implementar modelos propios.

---

# Capacidades de las Built-in Skills

## 1. Language Detection

Detecta automáticamente el idioma del texto.

Ejemplo:

```
Hola mundo

↓

language = es
```

---

## 2. Entity Recognition

Extrae entidades importantes.

Ejemplo:

Texto:

```
Microsoft abrió oficinas en Lima.
```

Resultado:

```
Empresa:
Microsoft

Ciudad:
Lima
```

---

## 3. Key Phrase Extraction

Extrae las frases más importantes del documento.

Ejemplo:

```
Azure AI Search
Vector Search
OpenAI
Knowledge Store
```

---

## 4. Translation

Traduce automáticamente el contenido.

Ejemplo:

```
Hello

↓

Hola
```

---

## 5. PII Detection

Detecta información personal (Personally Identifiable Information).

Ejemplos:

- DNI
- Pasaporte
- Teléfono
- Email
- Tarjeta de crédito

También puede eliminarla.

Ejemplo:

```
Juan Pérez
DNI 12345678

↓

Juan Pérez
DNI ********
```

---

## 6. OCR

Extrae texto contenido dentro de imágenes.

Ejemplo:

```
Imagen

↓

Texto
```

Muy útil para PDFs escaneados.

---

## 7. Image Captioning

Genera descripciones automáticas de imágenes.

Ejemplo:

```
Imagen

↓

"A dog playing in a park."
```

---

## 8. Image Tagging

Genera etiquetas automáticamente.

Ejemplo:

```
Imagen

↓

dog
park
grass
tree
animal
```

# Requisitos para usar Built-in Skills

Las Built-in Skills necesitan acceso a un recurso de **Azure AI Foundry Tools**.

Existen dos opciones.

---

## Opción 1

Usar el recurso gratuito incluido en Azure AI Search.

Limitación:

- máximo **20 documentos** para indexación con IA.

Ideal para pruebas o aprendizaje.

---

## Opción 2

Conectar un recurso de **Azure AI Foundry** propio.

Requisitos:

- Debe estar en la **misma región** que Azure AI Search.
- Sin la limitación de 20 documentos.

Es la opción recomendada para producción.

# Custom Skills

Si las Skills integradas no son suficientes, puedes crear tus propias **Custom Skills**.

Estas Skills ejecutan lógica personalizada y agregan nuevos campos al documento.

---

# ¿Cómo funcionan?

Generalmente se implementan como:

- Azure Functions
- APIs REST
- Servicios propios

El Indexer envía datos a la Custom Skill.

La Skill procesa la información y devuelve nuevos campos.

```
Documento

↓

Custom Skill

↓

Resultado personalizado

↓

Search Index
```

---

# Ejemplo con Azure Document Intelligence

Supongamos que indexas contratos.

Azure AI Search no extrae ciertos campos específicos.

Puedes crear una Azure Function que invoque **Azure Document Intelligence**.

Flujo:

```
Contrato PDF

↓

Indexer

↓

Azure Function

↓

Document Intelligence

↓

Campos extraídos

↓

Search Index
```

---

# Ejemplo de campos generados

Antes:

```
document
│
└── content
```

Después:

```
document
│
├── content
├── customer_name
├── contract_number
├── total_amount
└── expiration_date
```

Estos campos pueden ser buscados directamente.

# Ventajas de las Built-in Skills

- No requieren desarrollo.
- Fácil configuración.
- Integración directa con Azure AI Search.
- Aprovechan modelos de Azure AI.
- Escalables.
- Actualizadas por Microsoft.

---

# Ventajas de las Custom Skills

- Permiten implementar cualquier lógica.
- Integración con APIs externas.
- Integración con Azure Functions.
- Integración con modelos propios.
- Integración con otros servicios de Azure.
- Extracción de información específica del negocio.

---

|Mantenimiento|Microsoft|Usuario|

# Conceptos clave

- **Enrichment Pipeline:** Proceso que enriquece automáticamente los documentos durante la indexación.
- **Skillset:** Conjunto ordenado de AI Skills que ejecuta el Indexer.
- **Built-in Skills:** Habilidades predefinidas proporcionadas por Azure AI Search (OCR, traducción, detección de entidades, etc.).
- **Custom Skills:** Habilidades personalizadas que permiten integrar lógica propia o servicios externos, como Azure Document Intelligence.
- **Azure AI Foundry Tools:** Recurso necesario para ejecutar las Built-in Skills en escenarios de producción.
- **Documento enriquecido:** Documento final con información adicional (idioma, entidades, OCR, frases clave, embeddings, etc.) que será almacenada en el Search Index.
