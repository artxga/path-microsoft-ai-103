# Analyze images with Content Understanding

Azure Content Understanding permite analizar imágenes para extraer datos estructurados, identificar elementos visuales y generar descripciones mediante IA generativa.

---

# Formatos soportados

| Formato | Descripción |
|----------|-------------|
| JPEG | Fotografías estándar |
| PNG | Imágenes con transparencia |
| BMP | Imágenes bitmap |
| TIFF | Escaneos de alta calidad |
| HEIF | High Efficiency Image Format |
| PDF | Documentos de una o varias páginas con imágenes |

---

# Analizadores preconstruidos

| Analizador | Función |
|------------|----------|
| `prebuilt-image` | Análisis general de imágenes y generación de descripciones |
| `prebuilt-receipt` | Extrae información de recibos (vendedor, fecha, total, productos) |
| `prebuilt-invoice` | Extrae información de facturas (proveedor, montos, líneas, etc.) |
| `prebuilt-idDocument` | Extrae información de documentos de identidad |

---

# Crear un esquema de extracción (Field Schema)

Se define qué información queremos obtener de una imagen.

Cada campo puede utilizar uno de tres métodos:

| Método | Función | Ejemplo |
|---------|----------|----------|
| `extract` | Extrae el valor exactamente como aparece | Nombre de un producto |
| `classify` | Clasifica el contenido entre opciones definidas | Nuevo / Usado / Dañado |
| `generate` | Genera información mediante IA | Descripción de la escena |

---

## Ejemplo de esquema

```json
{
  "description": "Product image analyzer",
  "baseAnalyzerId": "prebuilt-image",
  "fieldSchema": {
    "fields": {
      "ProductName": {
        "type": "string",
        "method": "extract",
        "description": "Name of the product visible in the image"
      },
      "Condition": {
        "type": "string",
        "method": "classify",
        "description": "Condition of the product",
        "enum": [
          "new",
          "used",
          "damaged"
        ]
      },
      "Description": {
        "type": "string",
        "method": "generate",
        "description": "Brief description of what the image shows"
      }
    }
  }
}
```

# Analizar una imagen

```python
from azure.ai.contentunderstanding import ContentUnderstandingClient
from azure.ai.contentunderstanding.models import AnalysisInput, AnalysisResult
from azure.core.credentials import AzureKeyCredential
from azure.identity import DefaultAzureCredential
import json
import sys

# Crear cliente
credential = AzureKeyCredential(key)

client = ContentUnderstandingClient(
    endpoint=FOUNDRY_ENDPOINT,
    credential=credential,
    api_version="2025-11-01"
)

# Leer imagen
with open("my_image.png", "rb") as f:
    file_bytes = f.read()

try:

    poller = client.begin_analyze(
        analyzer_id=ANALYZER_ID,
        inputs=[
            AnalysisInput(data=file_bytes)
        ],
    )

    result: AnalysisResult = poller.result()

    print(json.dumps(result.as_dict(), indent=2))

except Exception as ex:
    print(f"[Unexpected Error]: {ex}")
    sys.exit(1)
```

# Resultado del análisis

La respuesta contiene principalmente:

## 1. markdown

Representación textual de toda la imagen.

Muy útil para:

- RAG
- Azure AI Search
- Indexación
- Búsquedas semánticas

---

## 2. fields

Los campos definidos en el schema.

Cada campo incluye:

- valor
- tipo
- confidence
- source (posición dentro de la imagen)

---

## 3. source (Grounding)

Indica exactamente de dónde salió el dato dentro de la imagen.

Ejemplo:

```
D(1,100,50,300,50,300,80,100,80)
```

Representa las coordenadas donde el modelo encontró ese valor.

Esto permite:

- auditoría
- explicabilidad
- revisión humana

# Ejemplo de respuesta

```json
{
  "contents": [
    {
      "markdown": "Product label showing 'Contoso Widget Pro' with serial number...",

      "fields": {

        "ProductName": {
          "type": "string",
          "valueString": "Contoso Widget Pro",
          "confidence": 0.95,
          "source": "D(1,100,50,300,50,300,80,100,80)"
        },

        "Condition": {
          "type": "string",
          "valueString": "new",
          "confidence": 0.89
        },

        "Description": {
          "type": "string",
          "valueString": "A silver electronic device in retail packaging with product label visible"
        }
      }
    }
  ]
}
```

# Confidence Score

Cada campo tiene un nivel de confianza entre **0 y 1**.

| Confidence | Interpretación |
|------------|----------------|
| **0.90 - 1.00** | Muy confiable. Puede automatizarse. |
| **0.70 - 0.89** | Moderadamente confiable. Conviene revisar en procesos críticos. |
| **< 0.70** | Baja confianza. Recomendada validación manual. |

Una estrategia común es:

```
Si confidence >= 0.90
    → Procesar automáticamente

Si confidence < 0.90
    → Enviar a revisión humana
```

# Buenas prácticas

- Utilizar imágenes de alta resolución.
- Mantener buena iluminación y contraste.
- Evitar imágenes con demasiados elementos.
- Mantener la orientación correcta (sin rotaciones).

# Flujo completo

```text
Imagen
   │
   ▼
Analyzer (prebuilt o custom)
   │
   ▼
OCR + Vision + IA Generativa
   │
   ▼
Extracción de campos
   │
   ├── markdown
   ├── fields
   ├── confidence
   └── source (grounding)
   │
   ▼
Resultado estructurado (JSON)
```
