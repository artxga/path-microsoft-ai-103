# What is Content Understanding?

**Azure Content Understanding** es una **Foundry Tool** que utiliza IA generativa para analizar contenido no estructurado y convertirlo en información estructurada y utilizable.

Puede procesar múltiples tipos de contenido:

- Documentos
- Imágenes
- Audio
- Video

Su objetivo es transformar información no estructurada en datos que puedan utilizarse en procesos de automatización, análisis o aplicaciones de IA.

---

# ¿Por qué usar Content Understanding?

## Simplifica el procesamiento

Permite procesar diferentes tipos de contenido mediante un flujo unificado.

En lugar de crear soluciones distintas para documentos, imágenes o audio, se utiliza un mismo servicio.

---

## Extracción sencilla de campos

Solo se define un **schema** indicando qué información se desea obtener.

No es necesario realizar ingeniería avanzada de prompts.

Ejemplo:

```
Factura

↓

Extraer:

- Número de factura
- Fecha
- Cliente
- Total
```

---

## Mayor precisión

Content Understanding utiliza múltiples modelos de IA para:

- analizar el contenido
- validar resultados
- mejorar la precisión

---

## Confidence Scores

Cada dato extraído incluye un nivel de confianza entre:

```
0.0
```

y

```
1.0
```

Ejemplo:

```json
{
    "invoiceNumber": {
        "value": "INV-1042",
        "confidence": 0.98
    }
}
```

---

## Grounding

Además del valor extraído, el servicio indica exactamente **de dónde proviene**.

Por ejemplo:

- página
- región del documento
- cuadro delimitador (bounding box)
- segmento de audio
- frame de video

Esto facilita la validación humana.

---

## Clasificación de contenido

Antes de extraer datos, el servicio puede clasificar el contenido.

Ejemplos:

- Factura
- Recibo
- Contrato
- Formulario
- Llamada de soporte

Luego puede enviarlo automáticamente al analizador adecuado.

---

# Componentes de Content Understanding

## 1. Inputs

Contenido de entrada.

Puede ser:

- Documentos
- Imágenes
- Audio
- Video

---

## 2. Analyzer

Es el componente principal.

Define:

- cómo procesar el contenido
- qué modelos utilizar
- qué campos extraer

---

## 3. Content Extraction

Convierte el contenido original en información entendible por la IA.

Puede incluir:

- OCR
- Speech-to-Text
- Detección de diseño (Layout Detection)

Resultado:

```
Contenido no estructurado

↓

Texto + Metadatos
```

---

## 4. Field Extraction

Utiliza el schema definido por el usuario para generar información estructurada.

Ejemplo:

```json
{
    "customer": "...",
    "invoiceDate": "...",
    "total": "..."
}
```

---

## 5. Confidence Scores

Cada campo recibe una puntuación de confianza.

Rango:

```
0 → 1
```

Ejemplo:

```
Invoice Number

Confidence: 0.99
```

---

## 6. Grounding

Permite identificar exactamente dónde fue encontrado cada dato.

Ejemplos:

- Página 3
- Línea 18
- Bounding Box
- Segmento del audio
- Frame del video

---

## 7. Structured Output

El resultado puede generarse en:

### Markdown

Ideal para:

- búsquedas
- RAG
- Azure AI Search

---

### JSON

Ideal para:

- APIs
- automatización
- aplicaciones empresariales

---

# Analizadores (Analyzers)

Son el componente más importante del servicio.

Definen cómo será procesado el contenido.

Existen dos tipos.

---

## Prebuilt Analyzers

Analizadores ya preparados por Microsoft.

Ejemplos:

- Facturas
- Recibos
- Call Centers

No requieren entrenamiento.

---

## Custom Analyzers

Analizadores creados por el usuario.

Se define:

- tipo de contenido
- schema de extracción
- configuración

Ejemplo:

```
Contrato

↓

Extraer:

- Cliente
- Empresa
- Fecha
- Cláusulas
- Monto
```

---

# Configuración de un Analyzer

Al crear un analyzer se configura:

- Tipo de contenido
- Modelos de IA
- Schema de extracción
- Confidence Scores
- Segmentación del contenido

---

# Casos de uso

## 1. Intelligent Document Processing

Procesar documentos automáticamente.

Ejemplos:

- Facturas
- Contratos
- Reclamos
- Formularios

---

## 2. Search y RAG

Convertir contenido multimodal para indexarlo en motores de búsqueda.

Ejemplos:

- Azure AI Search
- Sistemas RAG

---

## 3. Aplicaciones con Agentes

Transformar archivos complejos en información estructurada antes de entregarla al agente.

```
PDF

↓

Content Understanding

↓

JSON

↓

Agente
```

---

## 4. Analytics

Extraer datos estructurados para análisis y reportes.

---

# Restricciones de contenido

Content Understanding incorpora mecanismos de **Responsible AI**.

Utiliza **Azure AI Content Safety** para detectar contenido dañino.

Se filtran contenidos relacionados con:

- Violencia
- Discurso de odio
- Explotación

---

## Reconocimiento facial

Puede describir atributos faciales presentes en imágenes y videos.

---

## Datos biométricos

El procesamiento de datos biométricos requiere:

- consentimiento
- notificación adecuada

      ▼
Content Understanding

      ▼
Content Extraction

      │

Analyzer
      │
      ▼
Field Extraction
      │
      ▼
Confidence Scores
      │
      ▼

---

# Ideas clave para el examen

- **Azure Content Understanding** convierte contenido no estructurado en datos estructurados.
- Soporta:
  - Documentos
  - Imágenes
  - Audio
  - Video
- El componente principal es el **Analyzer**.
- Existen dos tipos de analyzers:
  - **Prebuilt**
  - **Custom**
- El procesamiento incluye:
  - OCR
  - Speech-to-Text
  - Layout Detection
  - Extracción de campos
- Cada campo incluye un **Confidence Score (0–1)**.
- **Grounding** indica la ubicación exacta del contenido del que se extrajo cada dato.
- Los resultados pueden obtenerse en **Markdown** (búsqueda/RAG) o **JSON** (automatización).
- Incluye protecciones de **Responsible AI** mediante **Azure AI Content Safety**.
