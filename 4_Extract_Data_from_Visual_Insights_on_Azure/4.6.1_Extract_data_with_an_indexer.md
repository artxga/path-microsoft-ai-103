# Extract Data with an Indexer (Azure AI Search)

En **Azure AI Search**, el componente encargado de **extraer, transformar e indexar** la información es el **Indexer**.

Su función es automatizar el proceso de leer datos desde una fuente, enriquecerlos mediante IA y construir el índice que posteriormente utilizarán las aplicaciones para realizar búsquedas.

---

# ¿Qué es un Indexer?

Un **Indexer** es un proceso automatizado que:

- Se conecta a una fuente de datos.
- Extrae los documentos.
- Procesa su contenido.
- Ejecuta AI Skills (si existen).
- Construye documentos enriquecidos.
- Actualiza automáticamente el Search Index.

No es necesario escribir código para importar documentos manualmente.

---

# Flujo general

```
Data Source
      │
      ▼
   Indexer
      │
      ▼
Document Cracking
      │
      ▼
Enrichment Pipeline
      │
      ▼
Search Index
```

---

# Data Source

El proceso comienza con una **fuente de datos**.

Ejemplos:

- Azure Blob Storage
- Azure SQL Database
- Azure Cosmos DB
- Azure Table Storage
- Otros orígenes compatibles

Los documentos originales permanecen en esta ubicación.

---

# Document Cracking

El primer paso del Indexer es realizar **Document Cracking**.

Consiste en abrir el documento y extraer todo su contenido.

Puede obtener:

- texto
- imágenes
- metadatos
- tablas
- estructura del documento

Por ejemplo, de un PDF puede extraer:

```
PDF

↓

Texto

↓

Imágenes

↓

Metadatos
```

---

# Enrichment Pipeline

Después de extraer el contenido, comienza el **Enrichment Pipeline**.

Aquí se ejecutan todas las AI Skills configuradas.

Ejemplos:

- OCR
- Language Detection
- Key Phrase Extraction
- Entity Recognition
- Embeddings
- Image Analysis
- Merge Skill

Cada skill agrega nueva información al documento.

---

# Construcción del documento

Durante la indexación, Azure AI Search construye un documento interno en formato **JSON**.

Inicialmente contiene únicamente los datos extraídos directamente del origen.

Ejemplo:

```
document
│
├── metadata_storage_name
├── metadata_author
└── content
```

---

# Documentos con imágenes

Si el documento contiene imágenes, el Indexer puede normalizarlas y almacenarlas en una colección llamada:

```
normalized_images
```

El documento pasa a tener esta estructura:

```
document
│
├── metadata_storage_name
├── metadata_author
├── content
└── normalized_images
      ├── image0
      └── image1
```

Cada imagen puede procesarse individualmente mediante AI Skills.

---

# Procesamiento jerárquico

El documento se organiza como un árbol jerárquico.

Esto permite aplicar una Skill sobre una parte específica del documento.

Ejemplo:

```
document
│
└── normalized_images
        │
        ├── image0
        └── image1
```

Una OCR Skill puede ejecutarse para **cada imagen** de la colección.

---

# Ejemplo: OCR

Después del OCR:

```
document
│
├── content
├── normalized_images
│      ├── image0
│      │      └── text
│      └── image1
│             └── text
```

Ahora el documento contiene el texto encontrado dentro de las imágenes.

---

# Agregar nuevos campos

Cada Skill puede agregar nuevos campos al documento.

Ejemplo:

Antes:

```
document
│
├── content
```

Después de Language Detection:

```
document
│
├── content
└── language
```

---

# Skills encadenadas

Las Skills pueden utilizar el resultado producido por otras Skills.

Ejemplo:

```
OCR

↓

Texto extraído

↓

Merge Skill

↓

Contenido combinado
```

---

# Merge Skill

Una Merge Skill puede unir:

- texto original
- texto obtenido mediante OCR

para crear un nuevo campo.

Resultado:

```
document
│
├── content
├── normalized_images
├── language
└── merged_content
```

El campo **merged_content** contiene todo el texto del documento, incluyendo el proveniente de las imágenes.

---

# Mapeo hacia el índice

Al finalizar el pipeline, el Indexer copia los datos al **Search Index**.

Existen dos formas de hacerlo.

---

## 1. Field Mapping

Se utiliza para campos provenientes directamente de la fuente de datos.

Puede ser:

### Implícito

Si el nombre coincide.

Ejemplo:

```
Blob

title

↓

Index

title
```

No requiere configuración.

---

### Explícito

Cuando deseas cambiar el nombre del campo o transformarlo.

Ejemplo:

```
Blob

document_name

↓

Index

title
```

---

# 2. Output Field Mapping

Se utiliza para mapear los resultados producidos por las AI Skills.

Ejemplo:

```
language

↓

Index.language
```

o

```
merged_content

↓

Index.fullText
```

Siempre debe configurarse explícitamente.

```

│
├── metadata

├── merged_content

├── key_phrases
├── embeddings
└── ...
```

Este documento es el que finalmente se almacena en el **Search Index**.

# Conceptos clave

- **Indexer:** Automatiza la extracción e indexación de documentos.
- **Document Cracking:** Extrae texto, imágenes y metadatos del documento original.
- **Enrichment Pipeline:** Ejecuta AI Skills para enriquecer el contenido.
- **normalized_images:** Colección donde se almacenan las imágenes extraídas para su análisis.
- **Field Mapping:** Copia campos desde la fuente de datos al índice.
- **Output Field Mapping:** Copia los resultados generados por las AI Skills al índice.
- **Search Index:** Índice final optimizado para realizar búsquedas rápidas y eficientes.
