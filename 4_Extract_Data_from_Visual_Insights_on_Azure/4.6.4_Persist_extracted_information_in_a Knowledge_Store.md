# Persist extracted information in a Knowledge Store

Aunque el **Search Index** es el resultado principal del proceso de indexación, los datos enriquecidos generados durante el pipeline pueden reutilizarse para otros escenarios.

Para ello, Azure AI Search ofrece **Knowledge Store**, un mecanismo que permite persistir la información enriquecida en distintos formatos para análisis, integración y automatización.

---

# ¿Qué es un Knowledge Store?

Un **Knowledge Store** es un almacenamiento donde Azure AI Search guarda una representación del contenido enriquecido generado durante el proceso de indexación.

En lugar de almacenar únicamente los documentos dentro del índice, también puede almacenar:

- Objetos JSON
- Tablas relacionales
- Archivos de imágenes

Todo ello se genera automáticamente durante la ejecución del **Indexer**.

---

# ¿Por qué usar un Knowledge Store?

El índice está optimizado para realizar búsquedas.

Sin embargo, muchas veces la información enriquecida también resulta útil para otros procesos.

Por ejemplo:

- Procesos ETL
- Business Intelligence
- Power BI
- Data Warehouse
- Machine Learning
- Auditoría
- Integraciones con otros sistemas

---

# Arquitectura

```text
Documentos

      │

      ▼
Indexer

      │

      ▼
Enrichment Pipeline
(AI Skills)

      │

 ┌───────────────┴───────────────┐
 │                               │
 ▼                               ▼

Search Index              Knowledge Store
```

El **Indexer** puede generar ambos resultados simultáneamente.

---

# ¿Qué almacena un Knowledge Store?

El Knowledge Store almacena **proyecciones** (*projections*).

Una proyección es una representación de los datos enriquecidos.

Puede tomar distintas formas.

---

# Tipos de proyecciones

## 1. Objetos JSON

Cada documento enriquecido puede almacenarse como un archivo JSON.

Ejemplo:

```json
{
  "title": "Azure AI Search",
  "language": "en",
  "author": "Microsoft",
  "entities": [
    "Azure",
    "AI",
    "Search"
  ]
}
```

Ideal para:

- ETL
- Integraciones
- APIs
- Data Lakes

---

## 2. Tablas

Los documentos también pueden normalizarse en tablas relacionales.

Ejemplo:

```text
Documents

+----+-------------------+-----------+

| ID | Title             | Language  |

+----+-------------------+-----------+

| 1  | Azure AI Search   | en        |

| 2  | Vision API        | es        |

+----+-------------------+-----------+
```

Permite realizar:

- consultas SQL
- reportes
- dashboards
- análisis estadístico

---

## 3. Archivos de imágenes

Si durante el proceso de indexación se extrajeron imágenes mediante OCR o Document Cracking, estas también pueden almacenarse.

```text
document.pdf

      │

      ▼

normalized_images

      │

      ▼

image1.png

image2.png

image3.png
```

Útil para:

- archivado
- entrenamiento de modelos
- inspección manual

---

# ¿Cómo se generan?

Durante la ejecución del **Indexer**.

```text
Data Source

      │

      ▼

Indexer

      │

      ▼

Skillset

      │

      ▼

Knowledge Store
```

No es necesario ejecutar un proceso adicional.

---

# Skillset

El **Knowledge Store** forma parte del **Skillset**.

Dentro del Skillset se definen las proyecciones que se desean almacenar.

```text
Skillset

│

├── OCR

├── Language Detection

├── Key Phrase Extraction

├── Entity Recognition

└── Knowledge Store
```

---

# Casos de uso

## ETL

Guardar los documentos enriquecidos como JSON para cargarlos posteriormente en un Data Lake.

---

## Business Intelligence

Persistir los datos en tablas para analizarlos mediante:

- SQL
- Power BI
- Azure Synapse

---

## Machine Learning

Guardar imágenes y anotaciones para entrenar modelos.

---

## Integraciones

Compartir los datos enriquecidos con otros sistemas empresariales.

---

# Beneficios

- Reutiliza toda la información generada por el pipeline.
- Evita repetir el procesamiento de IA.
- Facilita la integración con otros servicios.
- Permite crear soluciones analíticas.
- Se genera automáticamente durante la indexación.

---

# Search Index vs Knowledge Store

| Search Index | Knowledge Store |
|--------------|-----------------|
| Optimizado para búsquedas | Optimizado para almacenamiento y análisis |
| Consultado por aplicaciones | Consumido por procesos ETL y BI |
| Devuelve resultados de búsqueda | Persiste datos enriquecidos |
| Documentos JSON indexados | JSON, tablas e imágenes |
