# Develop a vision-based chat app

Para desarrollar una aplicación de chat con visión (multimodal), se utiliza prácticamente el mismo flujo que un chat tradicional. La diferencia es que el mensaje del usuario puede contener **texto + imágenes**.

## Flujo general

1. Conectarse al endpoint donde está desplegado el modelo.
2. Construir el prompt.
3. Enviar el prompt al modelo.
4. Procesar la respuesta.

La diferencia está en que el mensaje del usuario es **multipart**, incluyendo:

- Texto (`input_text` o `text`)
- Imagen (`input_image` o `image_url`)

---

# Prompt multimodal

Un mensaje del usuario puede contener varios elementos:

```
User Message
├── Texto
└── Imagen
```

Ejemplo conceptual:

```
Usuario:
    Texto: "¿Qué postres puedo preparar con esto?"
    Imagen: dragon-fruit.jpeg
```

El modelo analiza ambos elementos simultáneamente.

---

# Enviar imágenes usando Responses API (recomendado)

Las imágenes pueden enviarse de dos maneras:

- URL pública de una imagen.
- Imagen local codificada en Base64.

## Leer imagen local

```python
from pathlib import Path
import base64

image_path = Path("dragon-fruit.jpeg")
image_format = "jpeg"

with open(image_path, "rb") as image_file:
    image_data = base64.b64encode(image_file.read()).decode("utf-8")

data_url = f"data:image/{image_format};base64,{image_data}"
```

También podría usarse directamente una URL:

```text
https://mi-servidor.com/imagen.jpg
```

---

## Enviar prompt con Responses API

```python
response = client.responses.create(
    model="gpt-4.1",
    input=[
        {
            "role": "developer",
            "content": "You are an AI assistant for chefs planning recipes."
        },
        {
            "role": "user",
            "content": [
                {
                    "type": "input_text",
                    "text": "What desserts could I make with this?"
                },
                {
                    "type": "input_image",
                    "image_url": data_url
                }
            ]
        }
    ]
)

print(response.output_text)
```

### Observaciones

El contenido del usuario es una lista de partes:

```python
"content": [
    {
        "type": "input_text",
        "text": "..."
    },
    {
        "type": "input_image",
        "image_url": data_url
    }
]
```

Cada elemento representa un tipo de entrada diferente.

# Enviar imágenes usando Chat Completions API

Si el modelo no soporta Responses API (por ejemplo algunos modelos multimodales como Phi), se utiliza Chat Completions.

## Leer imagen

```python
from pathlib import Path
import base64

image_path = Path("orange.jpeg")
image_format = "jpeg"

with open(image_path, "rb") as image_file:
    image_data = base64.b64encode(image_file.read()).decode("utf-8")

data_url = f"data:image/{image_format};base64,{image_data}"
```

---

## Enviar prompt

```python
response = client.chat.completions.create(
    model="Phi-4-multimodal-instruct",
    messages=[
        {
            "role": "system",
            "content": "You are an AI assistant for chefs planning recipes."
        },
        {
            "role": "user",
            "content": [
                {
                    "type": "text",
                    "text": "What can I make with this fruit?"
                },
                {
                    "type": "image_url",
                    "image_url": {
                        "url": data_url
                    }
                }
            ]
        }
    ]
)

print(response.choices[0].message.content)
```

---

# Responses API vs Chat Completions

| Responses API | Chat Completions |
|---------------|------------------|
| API moderna | API clásica |
| `input` | `messages` |
| `input_text` | `text` |
| `input_image` | `image_url` |
| `response.output_text` | `response.choices[0].message.content` |
| Recomendada para modelos recientes | Usada por modelos que aún no soportan Responses |

---

# Formatos de imagen soportados

Puede enviarse una imagen mediante:

- URL pública
- Base64 (`data:image/...;base64,...`)

Ejemplo:

```text
data:image/jpeg;base64,/9j/4AAQSk...
```

o

```text
https://example.com/image.jpg
```

# Flujo completo

```
Imagen
      │
      ▼
Leer archivo
      │
      ▼
Codificar en Base64
      │
      ▼
Construir data:image/...;base64
      │
      ▼
Agregar al prompt
      │
      ▼
Responses API / Chat Completions
      │
      ▼
Modelo multimodal
      │
      ▼
Respuesta en texto
```

---

# Idea clave para el examen

- Un chat multimodal funciona igual que un chat tradicional.
- La diferencia es que el mensaje del usuario puede contener múltiples tipos de contenido (texto + imágenes).
- Las imágenes pueden enviarse mediante una URL pública o como Base64.
- Para modelos modernos se recomienda **Responses API**.
- Para modelos que no soportan Responses API se utiliza **Chat Completions API**.
