# What is Azure AI Search?

**Azure AI Search** es un servicio administrado en la nube que permite **indexar, enriquecer y consultar información** proveniente de múltiples fuentes de datos para construir soluciones de búsqueda escalables e inteligentes.

No solo indexa texto, sino que también puede utilizar **modelos de IA** para enriquecer automáticamente el contenido durante el proceso de indexación.

---

# ¿Qué permite hacer Azure AI Search?

- Indexar documentos y datos de múltiples orígenes.
- Consultar información mediante búsquedas rápidas y escalables.
- Enriquecer el contenido utilizando AI Skills.
- Almacenar la información enriquecida en un Knowledge Store.
- Construir soluciones de búsqueda empresarial.
- Implementar Retrieval-Augmented Generation (RAG).
- Crear soluciones de Knowledge Mining.

# Proceso de indexación

Durante la indexación, Azure AI Search puede:

1. Leer documentos.
2. Extraer texto.
3. Ejecutar modelos de IA.
4. Crear un índice optimizado para búsquedas.

```
Documento
     │
     ▼
Indexer
     │
     ▼
AI Skills
     │
     ▼
Search Index
```

---

# AI Enrichment

Uno de los mayores beneficios es el **AI Enrichment**, que consiste en aplicar modelos de IA automáticamente durante la indexación.

Ejemplos de enriquecimiento:

- OCR
- Detección de entidades
- Extracción de frases clave
- Detección de idioma
- Análisis de sentimiento
- Generación de embeddings
- Clasificación
- Resúmenes

Estos datos enriquecidos pasan a formar parte del índice de búsqueda.

---


---

# AI Skills

Las **AI Skills** son modelos que se ejecutan durante la indexación para enriquecer los documentos.

Algunos ejemplos:

- OCR
- Entity Recognition
- Key Phrase Extraction
- Language Detection
- Text Analytics
- Image Analysis
- Embeddings
- Azure OpenAI Skills

Estas habilidades forman parte del pipeline de indexación.

---

# Search Index

El **Search Index** es el componente principal del servicio.

Contiene la información optimizada para realizar búsquedas.

Puede incluir:

- texto original
- metadatos
- entidades
- tablas
- OCR
- embeddings
- vectores
- información enriquecida

Las aplicaciones consultan este índice, no los documentos originales.

---

# Knowledge Store

Además del índice, Azure AI Search puede guardar la información enriquecida en un **Knowledge Store**.

Su objetivo es reutilizar los datos para:

- Power BI
- análisis
- dashboards
- aplicaciones
- procesos ETL

```
Documento
      │
      ▼
AI Skills
      │
      ├────────► Search Index
      │
      ▼
Knowledge Store
```

# Casos de uso

## 1. Enterprise Search

Permite buscar información dentro de una organización.

Ejemplos:

- SharePoint
- PDFs
- Manuales
- Wikis
- Bases de conocimiento
- Documentación técnica

---

## 2. RAG (Retrieval-Augmented Generation)

Es uno de los usos más comunes actualmente.

Flujo:

```
Usuario

↓

Pregunta

↓

Azure AI Search

↓

Busca documentos relevantes

↓

LLM

↓

Respuesta fundamentada
```

En este escenario Azure AI Search actúa como el sistema de recuperación (Retriever).

---

## 3. Knowledge Mining

Consiste en descubrir información valiosa a partir de grandes volúmenes de documentos.

Ejemplos:

- contratos
- reportes
- correos
- facturas
- documentos legales

Durante la indexación se extraen automáticamente entidades, relaciones y otros datos útiles para análisis posteriores.

---

# Beneficios

- Servicio completamente administrado.
- Escalable.
- Búsquedas rápidas.
- Compatible con múltiples fuentes de datos.
- Integración con Azure AI Services.
- Soporte para búsqueda tradicional y vectorial.
- Ideal para aplicaciones RAG.
- Permite enriquecer documentos mediante IA sin necesidad de entrenar modelos propios.
