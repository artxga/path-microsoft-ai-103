# Search an Index

Una vez creado el índice en Azure AI Search, las aplicaciones cliente pueden realizar consultas para recuperar información.

Las consultas pueden:

- Buscar texto (Full-text Search)
- Filtrar resultados
- Ordenar resultados
- Obtener facetas (Facets)
- Seleccionar únicamente ciertos campos

---

# ¿Qué contiene un índice?

Un índice es una colección de documentos JSON.

Ejemplo:

```json
{
  "id": "1",
  "title": "Azure AI Search",
  "author": "Microsoft",
  "content": "Azure AI Search provides cloud search capabilities...",
  "language": "en",
  "last_modified": "2025-01-15"
}
```

Cada documento contiene los valores extraídos durante el proceso de indexación.

---

# Propiedades de un campo

Cada campo del índice puede configurarse con diferentes capacidades.

| Propiedad | Descripción |
|------------|-------------|
| key | Identificador único del documento. |
| searchable | Permite realizar búsquedas Full-text. |
| filterable | Permite aplicar filtros (`$filter`). |
| sortable | Permite ordenar resultados (`$orderby`). |
| facetable | Permite generar Facets para filtros dinámicos. |
| retrievable | Determina si el campo puede devolverse en los resultados. |

Ejemplo de definición:

```json
{
  "name": "author",
  "type": "Edm.String",
  "searchable": true,
  "filterable": true,
  "sortable": true,
  "facetable": true,
  "retrievable": true
}
```

---

# Full-text Search

La mayoría de las búsquedas utilizan **Full-text Search**, que analiza el contenido textual del índice para encontrar coincidencias relevantes.

Azure AI Search utiliza **Apache Lucene** como motor de búsqueda.

---

# Tipos de consultas

## 1. Simple Query

Es la sintaxis más sencilla y está orientada a búsquedas comunes realizadas por usuarios.

Ejemplo:

```text
hotel parking
```

Busca documentos que contengan esos términos.

---

## 2. Full Query

Permite consultas avanzadas utilizando toda la sintaxis de Lucene.

Permite:

- Expresiones complejas
- Expresiones regulares
- Operadores booleanos
- Prefijos
- Agrupaciones

Ejemplo:

```text
title:(hotel AND beach)
```

---

# Parámetros principales de búsqueda

| Parámetro | Función |
|-----------|----------|
| search | Texto que se desea buscar. |
| queryType | Tipo de consulta (`simple` o `full`). |
| searchFields | Campos donde buscar. |
| select | Campos que devolverá la consulta. |
| searchMode | Define cómo combinar varios términos. |

---

# Search Mode

## Any

Devuelve documentos que contengan cualquiera de los términos.

Consulta:

```text
comfortable hotel
```

Coinciden documentos con:

- comfortable
- hotel
- ambos

Equivale aproximadamente a:

```text
comfortable OR hotel
```

---

## All

Solo devuelve documentos que contengan todos los términos.

```text
comfortable hotel
```

Equivale aproximadamente a:

```text
comfortable AND hotel
```

---

# Flujo interno de una consulta

```text
Consulta

   │

   ▼
Query Parsing

   │

   ▼
Lexical Analysis

   │

   ▼
Document Retrieval

   │

   ▼
Scoring (TF-IDF)

   │

   ▼
Resultados
```

---

# 1. Query Parsing

La consulta se convierte en una estructura interna.

Ejemplos de subconsultas:

### Término

```text
hotel
```

Busca la palabra exacta.

---

### Frase

```text
"free parking"
```

Busca ambas palabras juntas.

---

### Prefijo

```text
air*
```

Puede coincidir con:

- airport
- airway
- air-conditioning

---

# 2. Lexical Analysis

El motor procesa lingüísticamente la consulta.

Realiza tareas como:

- convertir a minúsculas
- eliminar stopwords
- stemming
- separación de palabras compuestas

Ejemplo:

```text
Comfortable

↓

comfort
```

Stopwords eliminadas:

```text
the
a
is
of
```

---

# 3. Document Retrieval

Una vez procesados los términos, Azure AI Search busca documentos que contengan esas palabras.

```text
Índice

↓

Coincidencias

↓

Documentos candidatos
```

---

# 4. Relevance Scoring

Los documentos encontrados reciben un puntaje de relevancia utilizando **TF-IDF (Term Frequency / Inverse Document Frequency)**.

Los documentos con mayor puntuación aparecen primero.

---

# Filtrado de resultados

Los filtros permiten restringir los documentos devueltos.

Existen dos formas.

---

## Opción 1: Sintaxis Simple

```text
search=London+author='Reviewer'
queryType=Simple
```

Busca:

- documentos que contengan "London"
- cuyo autor sea "Reviewer"

---

## Opción 2: OData Filter

Mucho más flexible.

```text
search=London

$filter=author eq 'Reviewer'

queryType=Full
```

⚠️ Los filtros OData son **case-sensitive**.

---

# Ejemplos de filtros OData

Filtrar por autor:

```text
$filter=author eq 'John'
```

---

Filtrar por precio:

```text
$filter=price gt 100
```

---

Filtrar por fecha:

```text
$filter=year ge 2024
```

---

Filtrar múltiples condiciones:

```text
$filter=author eq 'John' and year ge 2024
```

---

# Facets

Los **Facets** permiten construir filtros dinámicos para la interfaz de usuario.

Ejemplo:

```text
search=*

facet=author
```

Respuesta:

```text
Microsoft (25)

Google (18)

OpenAI (9)

Amazon (6)
```

La aplicación puede mostrar estos valores como filtros.

---

Luego el usuario selecciona un autor.

```text
search=*

$filter=author eq 'Microsoft'
```

---

# Ordenamiento

Por defecto, Azure AI Search ordena por relevancia.

```text
TF-IDF Score
```

---

También puede ordenarse por cualquier campo marcado como `sortable`.

Ejemplo:

```text
search=*

$orderby=last_modified desc
```

Documentos más recientes primero.

---

Orden ascendente:

```text
$orderby=price asc
```

---

Orden descendente:

```text
$orderby=price desc
```

---

# Seleccionar campos

No siempre es necesario devolver todo el documento.

Puede utilizarse `select`.

Ejemplo:

```text
search=Azure

select=title,author
```

Respuesta:

```json
{
  "title": "Azure AI Search",
  "author": "Microsoft"
}
```

---

# Buscar únicamente en determinados campos

Con `searchFields` es posible limitar dónde buscar.

Ejemplo:

```text
search=Azure

searchFields=title
```

Solo buscará dentro del título.

---

# Ejemplo completo

Buscar documentos sobre Azure escritos por Microsoft, ordenados por fecha.

```text
search=Azure

queryType=Full

$filter=author eq 'Microsoft'

$orderby=last_modified desc

select=title,last_modified,author
```

---

```text

```

# Resumen

## Search Index

Colección de documentos JSON optimizados para búsqueda.

### Propiedades de un campo

- key
- searchable
- filterable
- sortable
- facetable
- retrievable

### Tipos de consultas

- Simple Lucene
- Full Lucene

### Pipeline de búsqueda

1. Query Parsing
2. Lexical Analysis
3. Document Retrieval
4. TF-IDF Scoring

### Funcionalidades adicionales

- Filtering (`$filter`)
- Facets (`facet`)
- Sorting (`$orderby`)
- Selección de campos (`select`)
- Búsqueda en campos específicos (`searchFields`)
