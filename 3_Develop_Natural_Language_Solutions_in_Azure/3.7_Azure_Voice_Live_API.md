# Azure Voice Live API

## Objetivo

Permite crear aplicaciones de voz con comunicación **bidireccional en tiempo real** mediante **WebSockets**.

Soporta:

- 🎤 Speech-to-Text
- 🔊 Text-to-Speech
- 🧑 Avatares (WebRTC)
- 🎧 Procesamiento de audio
- 💬 Conversaciones en tiempo real

---

# Características principales

- Comunicación en tiempo real mediante **WebSockets**.
- Eventos JSON entre cliente y servidor.
- Audio PCM16 y G.711.
- Voces OpenAI y voces personalizadas de Azure.
- Avatares con WebRTC.
- Reducción de ruido y cancelación de eco.

---

# Arquitectura

```text
Cliente
    │
    │ WebSocket
    ▼
Voice Live API
    │
    ├── Speech-to-Text
    ├── Text-to-Speech
    ├── Procesamiento de audio
    └── Avatar Streaming
```

# Autenticación

Dos opciones:

### Microsoft Entra ID (recomendado)

- Token Bearer
- Scope:
  - `https://ai.azure.com/.default`
  - (legacy) `https://cognitiveservices.azure.com/.default`

Header:

```http
Authorization: Bearer <token>
```

---

### API Key

Puede enviarse mediante:

- Header `api-key`
- Query parameter `api-key`

> En navegadores solo está disponible mediante **query string**.

# Eventos

## Cliente → Servidor

- session.update: Modify session configurations.
- input_audio_buffer.append: Add audio data to the buffer.
- response.create: Generate responses via model inference.

---

## Servidor → Cliente

- session.updated: Confirm session configuration changes.
- response.done: Indicate response generation completion.
- conversation.item.created: Notify when a new conversation item is added.

---

# Configurar la sesión

El primer evento suele ser:

```json
{
  "type": "session.update"
}
```

Permite configurar:

- modalidades (`text`, `audio`)
- voz
- instrucciones
- formato de audio
- frecuencia de muestreo
- detección de turnos (VAD)
- temperatura
- tokens máximos

Ejemplo:

```text
Modalidades: text + audio
Voz: alloy
Audio: PCM16
Sampling: 24 kHz
Temperature: 0.8
```

---

# Procesamiento de audio

Eventos disponibles:

- Append → agregar audio al buffer.
- Commit → procesar el buffer.
- Clear → limpiar el buffer.

Se puede activar:

- Reducción de ruido.
- Cancelación de eco.

Beneficios:

- Mejor calidad.
- Mayor precisión del reconocimiento.
- Conversaciones más naturales.

---

# Avatar Streaming

Utiliza **WebRTC**.

Evento:

```json
{
  "type": "session.avatar.connect"
}
```

Permite configurar:

- resolución
- bitrate
- codec
- blendshapes
- visemes
- animaciones faciales

# Flujo de funcionamiento

```text
Cliente
    │
    ▼
WebSocket
    │
    ▼
Voice Live API
    │
    ├── session.update
    ├── Audio Buffer
    ├── Speech Recognition
    ├── LLM
    ├── Text-to-Speech
    └── Avatar (opcional)
    │
    ▼
Respuesta en tiempo real
```

# Idea clave

La **Voice Live API** permite construir asistentes de voz conversacionales en tiempo real mediante **WebSockets**, gestionando audio, voz, reconocimiento de habla, síntesis de voz y avatares a través de eventos JSON.
