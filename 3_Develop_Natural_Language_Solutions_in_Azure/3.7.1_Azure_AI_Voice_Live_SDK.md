# Azure AI Voice Live SDK (Python)

## Objetivo

El **Azure AI Voice Live SDK** permite crear aplicaciones de voz en tiempo real utilizando la **Voice Live API**, proporcionando una interfaz asíncrona (`async/await`) sobre WebSockets.

> **Importante:** Desde la versión **1.0.0**, el SDK es **exclusivamente asíncrono**. La API síncrona fue deprecada.

---

# Características

- Comunicación en tiempo real mediante WebSockets.
- Streaming de audio (Speech-to-Speech).
- Manejo de eventos del servidor.
- Autenticación con API Key o Microsoft Entra ID.
- Configuración de sesiones de voz.

# Manejo de eventos

Los eventos llegan de forma continua desde el servidor y permiten reaccionar en tiempo real.

Ejemplo:

```python
async for event in connection:
    if event.type == ServerEventType.SESSION_UPDATED:
        print(f"Session ready: {event.session.id}")
        # Start audio capture

    elif event.type == ServerEventType.INPUT_AUDIO_BUFFER_SPEECH_STARTED:
        print("User started speaking")
        # Stop playback and cancel any current response

    elif event.type == ServerEventType.RESPONSE_AUDIO_DELTA:
        # Play the audio chunk
        audio_bytes = event.delta

    elif event.type == ServerEventType.ERROR:
        print(f"Error: {event.error.message}")
```

Eventos comunes:

| Evento | Descripción |
|---------|-------------|
| `SESSION_UPDATED` | La sesión quedó configurada correctamente. |
| `INPUT_AUDIO_BUFFER_SPEECH_STARTED` | El usuario comenzó a hablar. |
| `RESPONSE_AUDIO_DELTA` | Llegó un fragmento del audio generado. |
| `RESPONSE_DONE` | Finalizó la respuesta. |
| `ERROR` | Se produjo un error. |

> **Importante:** Cuando el usuario interrumpe al asistente, la aplicación debe detener inmediatamente la reproducción del audio para evitar que el asistente siga hablando mientras el usuario habla.

# Configurar una sesión

Una sesión se configura mediante un objeto `RequestSession`.

Permite definir:

- Modalidades (`TEXT`, `AUDIO`)
- Instrucciones del asistente
- Formato del audio
- Detección automática de turnos (VAD)

---

# Ejemplo completo

```python
import asyncio
from azure.core.credentials import AzureKeyCredential
from azure.ai.voicelive.aio import connect
from azure.ai.voicelive.models import (
    RequestSession,
    Modality,
    InputAudioFormat,
    OutputAudioFormat,
    ServerVad,
    ServerEventType
)

API_KEY = "your-api-key"
ENDPOINT = "your-endpoint"
MODEL = "gpt-4o"

async def main():
    async with connect(
        endpoint=ENDPOINT,
        credential=AzureKeyCredential(API_KEY),
        model=MODEL,
    ) as conn:

        session = RequestSession(
            modalities=[Modality.TEXT, Modality.AUDIO],
            instructions="You are a helpful assistant.",
            input_audio_format=InputAudioFormat.PCM16,
            output_audio_format=OutputAudioFormat.PCM16,
            turn_detection=ServerVad(
                threshold=0.5,
                prefix_padding_ms=300,
                silence_duration_ms=500
            ),
        )

        await conn.session.update(session=session)

        # Process events
        async for evt in conn:
            print(f"Event: {evt.type}")

            if evt.type == ServerEventType.RESPONSE_DONE:
                break

asyncio.run(main())
```

- abrir conexiones WebSocket,
- configurar sesiones,
- transmitir audio en tiempo real,
