# Connect and Use the Azure Language MCP Server

## Objetivo
Conectar el **Azure Language MCP Server** a un agente de **Microsoft Foundry** para que pueda utilizar herramientas de análisis de texto automáticamente.

## 1. Crear el agente

Requisitos:
- Proyecto en **Microsoft Foundry**.
- Modelo desplegado (ej. **GPT-4.1**).
- Crear un agente con instrucciones.

Ejemplo:

```text
You are an AI agent that assists users by helping them analyze and summarize text.
```

## 2. Conectar el Azure Language MCP Server

Desde **Foundry Portal → Tools**:

1. Connect a tool.
2. Seleccionar **Azure Language in Foundry Tools**.
3. Configurar:
   - Foundry Resource Name.
   - Autenticación por **API Key**.
   - Ocp-Apim-Subscription-Key.
4. Asociar la herramienta al agente.

Resultado:
- El agente obtiene acceso a todas las herramientas de Azure Language.

## 3. Actualizar las instrucciones

Indicar explícitamente que utilice la herramienta.

Ejemplo:

```text
You are an AI agent that assists users by helping them analyze text.
Use the Azure Language tool to perform text analysis tasks.
```

## 4. Probar en Agent Playground

Cuando el usuario envía un prompt:

1. El agente analiza la petición.
2. Decide qué herramientas necesita.
3. Invoca automáticamente los MCP Tools.
4. Devuelve una respuesta combinada.

La primera vez:
- Debe aprobarse el uso del MCP Tool.

Los **Logs** muestran:
- Herramienta utilizada.
- Entrada enviada.
- Resultado obtenido.

## 5. Crear una aplicación cliente

SDKs principales:

- `azure-ai-projects`
- `azure-identity`

Flujo:

```text
AIProjectClient
       │
       ▼
OpenAI Client
       │
       ▼
responses.create()
       │
       ▼
Agente
       │
       ▼
Azure Language MCP
       │
       ▼
Respuesta
```

Para llamar al agente:

- Obtener `OpenAIClient`.
- Ejecutar `responses.create()`.
- Referenciar el agente mediante `agent_reference`.

## 6. Conectar el MCP Server desde código

En lugar del portal, puede configurarse mediante `MCPTool`.

Se especifica:

- Server label.
- Server URL.
- Política de aprobación.
- Herramientas permitidas (`allowed_tools`, opcional).

Esto permite administrar la infraestructura desde código.

## 7. Prompts con múltiples tareas

Un mismo prompt puede utilizar varias herramientas.

Ejemplo:

```text
"Tell me what entities and dates are mentioned
and whether it is positive or negative."
```

El agente puede ejecutar:

- Named Entity Recognition
- Sentiment Analysis

Luego combina ambos resultados en una única respuesta.

## Flujo completo

```text
Usuario
    │
    ▼
Agente (Foundry)
    │
    ▼
Analiza el prompt
    │
    ▼
Selecciona herramientas MCP
    │
    ▼
Azure Language MCP Server
    │
    ├── Detect Language
    ├── NER
    ├── PII
    └── Health Analytics
    │
    ▼
Resultados
    │
    ▼
Respuesta final al usuario
```

## Idea clave

- El **Azure Language MCP Server** añade capacidades de análisis de texto a un agente de Foundry.
- El agente **selecciona automáticamente** las herramientas necesarias según el prompt.
- La conexión puede configurarse desde el **Foundry Portal** o mediante **`MCPTool`** en código.
- Un solo prompt puede provocar la ejecución de **múltiples herramientas MCP** en una misma interacción.
