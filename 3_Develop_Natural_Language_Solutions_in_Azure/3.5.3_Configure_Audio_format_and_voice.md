# Configure Audio Format and Voices

## ¿Qué se puede personalizar?
Al sintetizar voz con **Azure Speech**, el objeto **SpeechConfig** permite configurar:

- 🎵 Formato del audio.
- 🗣️ Voz utilizada.

## Audio Format

Define las características del audio generado.

Se puede configurar:

- Tipo de archivo.
- Sample Rate (frecuencia de muestreo).
- Bit Depth (profundidad de bits).

Ejemplo:

```python
speech_config.set_speech_synthesis_output_format(
    SpeechSynthesisOutputFormat.Riff24Khz16BitMonoPcm
)
```

### Ejemplo de formato

`Riff24Khz16BitMonoPcm`

Significa:

- **RIFF** → formato de archivo.
- **24 Khz** → frecuencia de muestreo.
- **16 Bit** → profundidad de bits.
- **Mono** → un canal de audio.
- **PCM** → audio sin compresión.

## Voices

Azure Speech ofrece múltiples voces.

Cada voz tiene un identificador con:

- Idioma/Región.
- Nombre.
- Tipo de voz.

Ejemplo:

```text
en-US-Brian:DragonHDLatestNeural
```

Desglose:

- `en-US` → Inglés (Estados Unidos).
- `Brian` → Voz.
- `DragonHDLatestNeural` → Modelo neuronal.

Configuración:

```python
speech_config.speech_synthesis_voice_name = \
    "en-US-Brian:DragonHDLatestNeural"
```

## Flujo

```text
SpeechConfig
      │
      ├── Audio Format
      └── Voice
              │
              ▼
SpeechSynthesizer
              │
              ▼
Audio personalizado
```

## Idea clave
- **SpeechConfig** permite personalizar la salida de voz.
- El **Audio Format** controla la calidad y el tipo del audio generado.
- La **Voice** define el idioma, el hablante y el modelo de voz utilizado.
