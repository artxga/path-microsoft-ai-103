# Microsoft Work IQ

## ¿Qué es Work IQ?

**Microsoft Work IQ** es una herramienta que permite conectar agentes de IA con los datos de **Microsoft 365 Copilot** mediante lenguaje natural.

Permite consultar información de la organización como:

- Correos electrónicos
- Reuniones
- Documentos
- Mensajes de Teams
- Personas y colaboradores

Su objetivo es proporcionar **contexto empresarial** para que los agentes generen respuestas más relevantes.

---

# Información que puede consultar

| Tipo de dato | Ejemplos |
|---------------|----------|
| **Emails** | Buscar correos o conversaciones con personas específicas |
| **Meetings** | Consultar calendario, notas y decisiones |
| **Documents** | Buscar archivos en SharePoint y OneDrive |
| **Teams** | Resumir conversaciones o encontrar mensajes |
| **People** | Identificar miembros del equipo o colaboradores |

---

# Work IQ y MCP

Work IQ está construido sobre **Model Context Protocol (MCP)**.

Actúa como un **MCP Server**, exponiendo capacidades que un agente puede descubrir y utilizar.

Estas capacidades incluyen:

- **Tools:** acciones (ej. buscar documentos).
- **Resources:** fuentes de datos de Microsoft 365.
- **Prompts:** plantillas predefinidas para consultas frecuentes.

```text
Agente
    │
    ▼
MCP Server (Work IQ)
    │
    ▼
Microsoft 365
```

---

# Modos de funcionamiento

## 1. CLI (Command Line Interface)

Permite realizar consultas directamente desde la terminal.

Ejemplo:

```bash
workiq ask -q "What requirements did Sarah share about the authentication feature?"
```

### Casos de uso

- Consultas rápidas.
- Scripts.
- Desarrollo.
- Automatización.

---

## 2. MCP Server

Work IQ funciona como un servidor MCP para asistentes de IA, como GitHub Copilot en Visual Studio Code.

El asistente puede consultar automáticamente información de Microsoft 365 sin que el usuario ejecute comandos manualmente.

Ejemplo:

```text
Usuario
    │
    ▼
GitHub Copilot
    │
    ▼
Work IQ (MCP Server)
    │
    ▼
Microsoft 365
```

### Casos de uso

- Desarrollo asistido por IA.
- Recuperación automática de contexto.
- Integración con agentes.

# Instalación

## Con npm

```bash
npm install -g @microsoft/workiq
```

O ejecutar sin instalar:

```bash
npx -y @microsoft/workiq mcp
```

---

## Como plugin de GitHub Copilot CLI

1. Abrir `copilot`.
2. Agregar el marketplace de plugins.
3. Instalar:

```text
workiq@copilot-plugins
```

---

## Configuración en Visual Studio Code

Agregar Work IQ como servidor MCP:

```json
{
  "workiq": {
    "command": "npx",
    "args": [
      "-y",
      "@microsoft/workiq",
      "mcp"
    ],
    "tools": [
      "*"
    ]
  }
}
```

Antes del primer uso:

```bash
workiq accept-eula
```

# Requisitos

Para utilizar Work IQ se necesita:

- Node.js.
- Suscripción Microsoft 365.
- Licencia Microsoft 365 Copilot.
- Consentimiento administrativo en Microsoft Entra ID.

---

# Seguridad

Work IQ sigue el mismo modelo de seguridad que Microsoft 365 Copilot.

## Acceso basado en permisos

Solo puede acceder a la información que el usuario ya tiene autorización para consultar.

---

## Sin almacenamiento

Work IQ **no almacena datos**.

Consulta la información bajo demanda.

---

## Seguridad empresarial

Las consultas respetan:

- Permisos de Microsoft Graph.
- Políticas de seguridad.
- Auditoría organizacional.

---

# Uso durante el desarrollo

## Opción 1: CLI

Ideal para consultas puntuales durante el desarrollo.

Ejemplos:

```bash
workiq ask -q "What were the key decisions in last week's architecture review meeting?"
```

```bash
workiq ask -q "Summarize the requirements in the user portal spec document"
```

```bash
workiq ask -q "What has the engineering team discussed about the API changes?"
```

---

## Opción 2: MCP Server

Cuando Work IQ está configurado como servidor MCP, el asistente de IA consulta automáticamente Microsoft 365 cuando detecta que necesita contexto.

Ejemplo:

```text
Usuario:
¿Qué requisitos compartió Sarah sobre la autenticación?

↓

GitHub Copilot

↓

Work IQ

↓

Microsoft 365

↓

Respuesta fundamentada
```

El usuario interactúa de forma natural con el asistente sin ejecutar comandos.

---

# CLI vs MCP Server

| Característica | CLI | MCP Server |
|----------------|-----|------------|
| Interfaz | Terminal | Asistente de IA |
| Consultas | Manuales | Automáticas |
| Ideal para | Scripts y consultas rápidas | Desarrollo asistido por IA |
| Integración | Baja | Alta |

---

# Resumen

- **Work IQ** conecta agentes con datos de **Microsoft 365 Copilot**.
- Funciona como un **MCP Server**, proporcionando herramientas y recursos para consultar información empresarial.
- Puede utilizarse de dos formas:
  - **CLI:** consultas manuales desde la terminal.
  - **MCP Server:** integración transparente con asistentes como GitHub Copilot.
- Respeta los permisos de Microsoft 365, no almacena datos y utiliza Microsoft Graph para acceder a la información autorizada.
