# Opciones de publicación para Microsoft 365

## ¿Por qué publicar un agente?

Después de desarrollar y probar un agente en **Microsoft AI Foundry**, el siguiente paso es publicarlo para que los usuarios puedan interactuar con él desde herramientas como **Microsoft Teams** y **Microsoft 365 Copilot**.

Publicar un agente permite llevarlo desde un entorno de desarrollo a un entorno donde otros usuarios puedan utilizarlo de forma segura y gobernada. :contentReference[oaicite:0]{index=0}

---

# Opciones de publicación

Microsoft Foundry ofrece dos formas principales de publicar un agente.

## 1. Publicación directa (Direct Publish)

Foundry realiza automáticamente el proceso de publicación hacia Microsoft 365.

Durante este proceso:

- Crea la aplicación del agente.
- Configura la integración con Teams y Microsoft 365 Copilot.
- Gestiona la infraestructura necesaria.

Es la opción recomendada para la mayoría de escenarios.

### Ventajas

- Publicación con pocos pasos.
- No requiere configurar manualmente manifiestos.
- Integración automática con Microsoft 365.

---

## 2. Descargar el manifiesto (Download Manifest)

En lugar de publicar directamente, puedes descargar el **manifiesto del agente**.

Esto permite:

- Personalizar la configuración.
- Modificar metadatos.
- Realizar una instalación manual (*sideload*) en Microsoft Teams.
- Distribuir el paquete mediante otros mecanismos.

Es útil cuando se necesita un mayor nivel de personalización. :contentReference[oaicite:1]{index=1}

---

# Alcances de publicación (Publish Scope)

Al utilizar **Direct Publish**, se puede elegir quién tendrá acceso al agente.

## 1. Just you (Individual Scope)

El agente queda disponible únicamente para el propietario.

### Características

- Disponible inmediatamente.
- No requiere aprobación de un administrador.
- Puede compartirse mediante un enlace.

### Ideal para

- Desarrollo.
- Pruebas.
- Pilotos.
- Equipos pequeños.

---

## 2. People in your organization (Organization Scope)

El agente se publica para toda la organización.

### Características

- Requiere aprobación del administrador de Microsoft 365.
- Una vez aprobado, aparece como un agente corporativo para todos los usuarios autorizados.
- Permite una distribución centralizada.

### Ideal para

- Producción.
- Implementaciones empresariales.
- Uso organizacional. :contentReference[oaicite:2]{index=2}

---

# Comparación

| Opción | Administración | Mejor para |
|---------|---------------|------------|
| **Direct Publish** | Publicación automática desde Foundry | La mayoría de escenarios |
| **Download Manifest** | Configuración e instalación manual | Personalización avanzada |

| Scope | Requiere aprobación | Visibilidad |
|--------|---------------------|-------------|
| **Just you** | ❌ No | Solo el propietario (se puede compartir mediante enlace) |
| **People in your organization** | ✅ Sí | Usuarios de toda la organización |

# Buenas prácticas

- Probar completamente el agente antes de publicarlo.
- Utilizar **Just you** durante el desarrollo y las pruebas.
- Utilizar **Organization Scope** para despliegues empresariales.
- Elegir **Download Manifest** solo cuando se necesite personalizar la distribución o el manifiesto del agente.

---

# Resumen

- Microsoft AI Foundry permite publicar agentes directamente en **Microsoft Teams** y **Microsoft 365 Copilot**.
- Existen dos métodos de publicación:
  - **Direct Publish:** publicación automática (recomendada).
  - **Download Manifest:** descarga del paquete para personalización o instalación manual.
- Con **Direct Publish** se puede elegir el alcance del agente:
  - **Just you:** para pruebas y uso personal.
  - **People in your organization:** para distribución organizacional con aprobación del administrador.
