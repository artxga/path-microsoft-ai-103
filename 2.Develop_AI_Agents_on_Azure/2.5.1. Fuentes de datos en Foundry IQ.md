# Fuentes de datos en Foundry IQ

## ¿Qué son las Knowledge Sources?

Las **Knowledge Sources** son las fuentes de información que alimentan una **Knowledge Base** en Foundry IQ.

Una Knowledge Base puede conectarse a **una o varias fuentes de datos**, permitiendo que los agentes consulten información distribuida en distintos sistemas. :contentReference[oaicite:0]{index=0}

---

# Tipos de fuentes de datos

Foundry IQ admite dos grandes categorías de fuentes de datos:

## 1. Fuentes indexadas (Indexed Sources)

Los datos se copian e indexan en **Azure AI Search**.

Durante este proceso Foundry IQ:

- Divide los documentos (*chunking*).
- Genera embeddings.
- Extrae metadatos.
- Crea un índice para búsquedas rápidas.

### Ventajas

- Búsquedas muy rápidas.
- Soporta búsqueda semántica, vectorial e híbrida.
- Permite actualizar el índice automáticamente.

### Ideal para

- Documentación técnica.
- PDFs.
- Manuales.
- Bases de conocimiento.
- Grandes volúmenes de documentos.

---

## 2. Fuentes remotas (Remote Sources)

Los datos **no se indexan**.

Cuando el agente recibe una consulta, Foundry IQ accede directamente a la fuente original para recuperar la información.

### Ventajas

- Siempre consulta la información más reciente.
- No requiere mantener un índice.

### Consideraciones

- El rendimiento depende del sistema remoto.
- La búsqueda puede ser más lenta que en fuentes indexadas.

---

# Orígenes compatibles

## Azure Blob Storage

Permite almacenar documentos como:

- PDF
- Word
- Markdown
- Texto
- Archivos de documentación

Es una de las fuentes más utilizadas para documentación empresarial.

---

## SharePoint

Permite consultar:

- Documentos corporativos.
- Manuales.
- Políticas.
- Procedimientos.

Muy útil para organizaciones que utilizan Microsoft 365.

---

## OneLake

Permite acceder a datos almacenados en Microsoft Fabric.

Ideal para:

- Datos empresariales.
- Información analítica.
- Grandes volúmenes de información.

---

## Azure AI Search

Puede utilizar índices existentes de Azure AI Search como fuente de conocimiento.

Esto permite reutilizar índices ya creados sin volver a procesar los documentos.

---

## Sitios web públicos

Foundry IQ puede utilizar páginas web públicas como fuente de información.

Útil para:

- Documentación oficial.
- Sitios corporativos.
- Información pública.

---

# Elegir el tipo de fuente

| Fuente | ¿Se indexa? | Mejor para |
|---------|-------------|------------|
| Azure Blob Storage | ✅ Sí | Documentos y archivos |
| SharePoint | ✅ Sí | Documentación empresarial |
| OneLake | ✅ Sí | Datos de Microsoft Fabric |
| Azure AI Search | ✅ Sí | Índices existentes |
| Sitios web | Puede ser indexada o consultada remotamente | Información pública |
| Fuente remota | ❌ No | Datos que cambian constantemente |

---

# ¿Cómo utiliza Foundry IQ estas fuentes?

```text
Knowledge Base
        │
        ├── Azure Blob Storage
        ├── SharePoint
        ├── OneLake
        ├── Azure AI Search
        └── Sitios web
                │
                ▼
        Agentic Retrieval
                │
                ▼
        Información relevante
                │
                ▼
        Respuesta del agente
```

Una misma **Knowledge Base** puede combinar múltiples fuentes de datos y decidir automáticamente cuál consultar para responder una pregunta. :contentReference[oaicite:1]{index=1}

# Resumen

- Las **Knowledge Sources** son las fuentes de información de una **Knowledge Base**.
- Foundry IQ admite **fuentes indexadas** y **fuentes remotas**.
- Entre las fuentes compatibles se encuentran:
  - Azure Blob Storage.
  - SharePoint.
  - OneLake.
  - Azure AI Search.
  - Sitios web públicos.
- Una Knowledge Base puede combinar varias fuentes para ofrecer respuestas más completas y fundamentadas. :contentReference[oaicite:2]{index=2}
