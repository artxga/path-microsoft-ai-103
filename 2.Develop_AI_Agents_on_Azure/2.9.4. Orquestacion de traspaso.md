# Orquestación de Traspaso (Handoff Orchestration)

La **orquestación de traspaso** (*Handoff Orchestration*) permite que los agentes de IA se transfieran el control entre sí de manera secuencial en función del contexto de la tarea o de las solicitudes del usuario. Cada agente puede delegar la conversación a otro que posea la experiencia adecuada, garantizando que el especialista más apto maneje cada parte del proceso.

A diferencia de los patrones paralelos, los agentes trabajan uno a la vez, entregando por completo el control al siguiente. Es un patrón ideal para soporte al cliente, sistemas expertos o cualquier escenario donde se requiera delegación dinámica.

---

## ¿Cuándo utilizar la orquestación de traspaso?

* **Conocimiento especializado:** Las tareas requieren herramientas o conocimientos específicos, pero el número o el orden de los agentes no se puede determinar de antemano.
* **Surgimiento dinámico de requisitos:** Las necesidades de experiencia aparecen durante el procesamiento, lo que activa el enrutamiento de tareas basado en el análisis del contenido.
* **Problemas multidominio:** Se requiere que diferentes especialistas trabajen de forma secuencial.
* **Reglas de transferencia claras:** Es posible definir señales o reglas precisas que indiquen cuándo un agente debe transferir el control y a quién.

## ¿Cuándo evitarla?

* **Flujos fijos:** Los agentes involucrados y su orden ya se conocen de antemano y son fijos.
* **Enrutamiento simple:** El enrutamiento de tareas es básico y basado en reglas estáticas, sin necesidad de interpretación dinámica.
* **Impacto al usuario:** Las decisiones erróneas de enrutamiento pueden frustrar a los usuarios.
* **Operaciones simultáneas:** Múltiples operaciones deben ejecutarse al mismo tiempo.
* **Riesgo de bucles:** Es difícil evitar bucles infinitos de traspaso o un rebote excesivo entre agentes.

---

## Implementación en el Microsoft Agent Framework SDK

Este patrón se implementa utilizando **flujos de trabajo de control** (*control workflows*). Cada agente procesa la tarea en secuencia y, en función de su salida, el flujo de trabajo decide a qué agente llamar a continuación mediante una estructura *switch-case*.

### 1. Configuración de Modelos de Datos y Cliente de Chat
* Crear el cliente de chat para conectarse a los servicios de IA.
* Definir modelos **Pydantic** para obtener respuestas JSON estructuradas de los agentes de IA.
* Crear clases de datos simples (*data classes*) para pasar información entre los pasos del flujo de trabajo.
* Configurar los agentes con instrucciones específicas y el parámetro `response_format` para salidas JSON estructuradas.

### 2. Creación de Ejecutores Especializados (*Executors*)
* **Ejecutor de almacenamiento de entrada:** Guarda los datos entrantes en el estado compartido y los reenvía al agente de clasificación.
* **Ejecutor de transformación:** Convierte la respuesta JSON del agente en un objeto de enrutamiento tipado.
* **Ejecutores manejadores (*Handlers*):** Ejecutores separados para cada resultado de clasificación, con condiciones de guarda para verificar el procesamiento correcto de los mensajes.

### 3. Construcción de la Lógica de Enrutamiento
* Crear funciones de fábrica (*factory functions*) que generen verificadores de condiciones para cada valor de clasificación.
* Diseñar condiciones que examinen los mensajes entrantes y devuelvan `true` para resultados de clasificación específicos.
* Utilizar las condiciones con objetos `Case` en grupos de bordes *switch-case*.
* **Siempre** incluir un caso `Default` como respaldo para escenarios inesperados.

### 4. Ensamblaje del Flujo de Trabajo
* Utilizar `WorkflowBuilder` para conectar los ejecutores con bordes regulares.
* Añadir el grupo de bordes *switch-case* para el enrutamiento basado en los resultados de la clasificación.
* Configurar el flujo de trabajo para seguir el primer caso coincidente o recurrir al caso por defecto.
* Configurar un ejecutor terminal para entregar el resultado final.

---

> **Conclusión:** La orquestación de traspaso proporciona una forma flexible y adaptable de enrutar tareas dinámicamente entre agentes de IA especializados. Al utilizar el Microsoft Agent Framework SDK, se pueden construir sistemas capaces de transferir el control fluidamente entre agentes e incluso integrar la intervención humana cuando sea necesario.
