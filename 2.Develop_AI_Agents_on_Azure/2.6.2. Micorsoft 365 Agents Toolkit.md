# Microsoft 365 Agents Toolkit (Avanzado)

> **Nota:** Para la mayoría de escenarios, Microsoft recomienda publicar el agente **directamente desde Microsoft AI Foundry**. El **Microsoft 365 Agents Toolkit** está pensado para escenarios empresariales avanzados que requieren mayor personalización.

---

# ¿Qué es el Microsoft 365 Agents Toolkit?

Es un conjunto de herramientas disponible como extensión para:

- Visual Studio Code
- Visual Studio

Permite crear una **aplicación proxy** entre Microsoft Teams/Microsoft 365 Copilot y un agente de Microsoft AI Foundry.

```text
Teams / Microsoft 365 Copilot
            │
            ▼
 Proxy App (Agents Toolkit)
            │
            ▼
      Foundry Agent
```

La aplicación proxy recibe las solicitudes, puede procesarlas mediante lógica personalizada y luego las envía al agente de Foundry.

---

# ¿Cuándo utilizarlo?

Utiliza el **Agents Toolkit** cuando necesites funcionalidades que la publicación directa desde Foundry no ofrece.

## Casos de uso

### Single Sign-On (SSO) personalizado

Cuando la autenticación requiere una configuración distinta a la predeterminada de Microsoft Entra ID.

---

### Middleware personalizado

Permite ejecutar lógica adicional antes o después de llamar al agente.

Ejemplos:

- Validaciones.
- Transformación de datos.
- Logging.
- Auditoría.
- Enrutamiento de solicitudes.

---

### Múltiples entornos

Facilita administrar distintos entornos como:

- Desarrollo
- Staging
- Producción

Cada uno con configuraciones independientes.

---

### Depuración avanzada

Permite utilizar las herramientas del IDE para:

- Breakpoints.
- Debug paso a paso.
- Trazas detalladas.
- Inspección de variables.

---

### Integración con CI/CD

Se integra con herramientas como:

- GitHub Actions
- Azure DevOps

para automatizar despliegues.

---

# ¿Cómo funciona?

En lugar de publicar directamente el agente, se crea una aplicación intermediaria.

```text
Usuario
      │
      ▼
Microsoft Teams / Copilot
      │
      ▼
Azure Bot Service
      │
      ▼
Proxy App (Agents Toolkit)
      │
      ├── Middleware personalizado
      ├── Logging
      ├── Transformaciones
      └── Autenticación
      │
      ▼
Foundry Agent
      │
      ▼
Respuesta
```

La ventaja es que tienes control total sobre el flujo de mensajes.

La desventaja es una mayor complejidad de desarrollo y mantenimiento.

---

# Flujo básico de configuración

1. Instalar la extensión **Microsoft 365 Agents Toolkit** en Visual Studio Code.
2. Crear un nuevo proyecto.
3. Seleccionar **Custom Engine Agent**.
4. Configurar el endpoint del agente de Foundry.
5. Configurar la autenticación.
6. Agregar middleware si es necesario.
7. Probar el agente localmente con **Microsoft 365 Agents Playground**.
8. Desplegar la aplicación proxy en Azure.
9. Registrar la aplicación en Microsoft Teams.

---

# Microsoft 365 Agents Playground

El Toolkit incluye un entorno local para pruebas.

Permite:

- Simular Microsoft Teams.
- Enviar mensajes al agente.
- Validar autenticación.
- Verificar el middleware antes del despliegue.

# Comparación

| Aspecto | Publicación directa | Agents Toolkit |
|----------|---------------------|----------------|
| Tiempo de configuración | Minutos | Horas o días |
| Código necesario | No | Sí (Proxy App) |
| Personalización | Limitada | Muy alta |
| Depuración | Foundry Portal | IDE completo |
| Mejor para | Escenarios estándar | Escenarios empresariales complejos |

---

# ¿Cuál elegir?

### Publicación directa desde Foundry

Úsala cuando:

- Quieras publicar rápidamente.
- No necesites middleware.
- No requieras autenticación personalizada.
- Tengas un escenario estándar.

---

### Microsoft 365 Agents Toolkit

Úsalo cuando:

- Necesites SSO personalizado.
- Requieras middleware.
- Tengas múltiples entornos.
- Necesites depuración avanzada.
- Quieras integrar el despliegue con CI/CD.

---

# Resumen

- El **Microsoft 365 Agents Toolkit** permite integrar un agente de Foundry mediante una **aplicación proxy**.
- Está orientado a escenarios empresariales que requieren mayor control sobre autenticación, procesamiento y despliegue.
- Ofrece funcionalidades como middleware personalizado, depuración avanzada, soporte para múltiples entornos e integración con pipelines de CI/CD.
- Para la mayoría de aplicaciones, Microsoft recomienda utilizar la **publicación directa desde Microsoft AI Foundry**, ya que es más simple y rápida.
