# Implement an Agent Executor

## ¿Qué es?
- Componente central de un agente **A2A**.
- Conecta el protocolo A2A con la lógica de negocio del agente.
- Procesa solicitudes y devuelve respuestas.

## Responsabilidades
- ▶️ Ejecutar tareas solicitadas.
- 📤 Enviar respuestas o eventos al cliente.
- ❌ Gestionar cancelaciones (si están soportadas).

## Operaciones principales

### Execute
Procesa una solicitud.

**Acciones:**
- Recibe el `RequestContext`.
- Ejecuta la lógica del agente.
- Envía resultados mediante el `EventQueue`.

### Cancel
- Cancela una tarea en ejecución.
- Es opcional; agentes simples pueden no implementarlo.

## Componentes clave

### RequestContext
Contiene la información de la solicitud:
- Entrada del usuario.
- Contexto de la tarea.
- Metadatos.

### EventQueue
Canal por el que el agente comunica:
- Mensajes.
- Actualizaciones de progreso.
- Resultados (artifacts).

## Flujo de ejecución
```text
Solicitud
    ↓
Agent Executor
    ↓
Lógica del agente
    ↓
EventQueue
    ↓
Cliente / Otro agente
```

## Ejemplo (Hello World)
1. Llega una solicitud.
2. El **Agent Executor** invoca la lógica del agente.
3. Se genera `"Hello World"`.
4. La respuesta se coloca en el **EventQueue**.
5. El cliente recibe el resultado.

## Idea clave
- **Agent Executor = ejecuta la lógica del agente.**
- **RequestContext = entrada de la solicitud.**
- **EventQueue = salida de respuestas y eventos.**
- Es el componente que hace que un agente A2A pueda recibir solicitudes y responder de forma estandarizada.
