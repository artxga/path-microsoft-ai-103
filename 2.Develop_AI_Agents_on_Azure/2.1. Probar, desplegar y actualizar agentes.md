# Probar, desplegar e integrar agentes

Una vez desarrollado un agente en **Microsoft AI Foundry**, el siguiente paso es **probar su funcionamiento**, **desplegarlo** e **integrarlo** con aplicaciones o servicios externos.

---

# 1. Probar el agente

Antes del despliegue, es importante verificar que el agente:

- Responda correctamente a los prompts.
- Utilice las herramientas (*Tools*) cuando corresponda.
- Llame a las funciones adecuadas.
- Mantenga el contexto de la conversación.
- Genere respuestas seguras y relevantes.

Las pruebas ayudan a detectar errores antes de publicar el agente.

---

# Métodos de prueba

## Chat Playground

Permite interactuar con el agente desde una interfaz de chat para validar su comportamiento.

Ideal para:

- Probar prompts.
- Verificar respuestas.
- Comprobar el uso de herramientas.
- Detectar errores rápidamente.

---

## Evaluación iterativa

Después de cada cambio se recomienda volver a probar el agente.

El ciclo es:

```text
Modificar agente
      │
      ▼
Probar
      │
      ▼
Analizar resultados
      │
      ▼
Mejorar
      │
      ▼
Repetir
```

---

# 2. Desplegar el agente

Una vez validado, el agente puede publicarse para que otras aplicaciones puedan utilizarlo.

El despliegue crea un **endpoint** accesible mediante API.

Las aplicaciones pueden enviar solicitudes al agente y recibir respuestas sin necesidad de acceder al entorno de desarrollo.

---

# 3. Integración

Después del despliegue, el agente puede integrarse con distintos tipos de aplicaciones.

Ejemplos:

- Aplicaciones web.
- Aplicaciones móviles.
- APIs.
- Microsoft Teams.
- Automatizaciones.
- Sistemas empresariales.

La comunicación se realiza mediante llamadas al endpoint del agente.

---

# Flujo completo

```text
Usuario
      │
      ▼
Aplicación
      │
      ▼
Endpoint del agente
      │
      ▼
Agente
      │
      ├── Modelo
      ├── Tools
      ├── Function Calling
      └── RAG (opcional)
      │
      ▼
Respuesta
```

---

# Buenas prácticas

- Probar distintos escenarios antes del despliegue.
- Validar el uso correcto de las herramientas.
- Comprobar que las respuestas sean seguras y precisas.
- Realizar pruebas después de cada cambio importante.
- Supervisar el comportamiento del agente una vez publicado.

---

# Resumen

- Antes de publicar un agente, debe probarse mediante el **Chat Playground** y pruebas iterativas.
- El despliegue crea un **endpoint** para que otras aplicaciones puedan comunicarse con el agente.
- Los agentes pueden integrarse con aplicaciones web, móviles, APIs y otros servicios.
- El proceso recomendado es **desarrollar → probar → desplegar → integrar → supervisar**, realizando mejoras continuas según sea necesario.
