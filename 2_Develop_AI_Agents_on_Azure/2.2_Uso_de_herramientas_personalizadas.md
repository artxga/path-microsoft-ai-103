# Uso de herramientas personalizadas (Custom Tools)

## ¿Qué son las Custom Tools?

Las **Custom Tools** son herramientas creadas por el desarrollador que permiten a un agente interactuar con sistemas externos o ejecutar lógica personalizada.

A diferencia de las herramientas integradas (como **Web Search**, **File Search** o **Code Interpreter**), las Custom Tools se adaptan a las necesidades específicas de una aplicación.

---

# ¿Para qué sirven?

Permiten que un agente pueda:

- Consultar APIs externas.
- Acceder a bases de datos.
- Ejecutar lógica de negocio.
- Obtener información en tiempo real.
- Integrarse con sistemas empresariales.
- Automatizar procesos.

---

# ¿Cómo funcionan?

```text
Usuario
    │
    ▼
Pregunta
    │
    ▼
Agente
    │
¿Necesita una herramienta?
    │
 ┌──┴─────────────┐
 │ Sí            │ No
 ▼               ▼
Custom Tool   Respuesta directa
 │
 ▼
Sistema externo / API
 │
 ▼
Resultado
 │
 ▼
Agente
 │
 ▼
Respuesta final
```

El agente decide cuándo utilizar una herramienta y utiliza el resultado para construir la respuesta.

---

# Tipos de Custom Tools

## Azure Functions

Permiten ejecutar código sin administrar servidores (**serverless**).

Casos de uso:

- Consultar APIs.
- Procesar datos.
- Ejecutar lógica personalizada.
- Automatizar tareas.

---

## Azure Logic Apps

Permiten crear flujos de trabajo sin escribir mucho código.

Ejemplos:

- Enviar correos.
- Crear aprobaciones.
- Integrarse con Microsoft 365.
- Automatizar procesos empresariales.

---

## APIs REST

El agente puede consumir cualquier API HTTP.

Ejemplos:

- Clima.
- Inventario.
- Pagos.
- CRM.
- ERP.

---

# Flujo de uso

1. El usuario realiza una consulta.
2. El agente determina que necesita información externa.
3. Selecciona la herramienta adecuada.
4. Ejecuta la herramienta.
5. Recibe el resultado.
6. Genera la respuesta final utilizando esa información.

---

# Ejemplo

**Usuario**

```text
¿Cuántas unidades quedan del producto X?
```

**Proceso**

```text
Agente
      │
      ▼
Custom Tool
(getInventory)
      │
      ▼
Base de datos
      │
      ▼
25 unidades
      │
      ▼
Respuesta al usuario
```

---

# Beneficios

- Amplían las capacidades del agente.
- Permiten acceder a información actualizada.
- Integran IA con sistemas empresariales.
- Automatizan tareas.
- Evitan limitar al modelo a su conocimiento de entrenamiento.

---

# Buenas prácticas

- Diseñar herramientas con una única responsabilidad.
- Escribir descripciones claras para que el agente sepa cuándo utilizarlas.
- Validar los parámetros de entrada.
- Manejar errores de forma adecuada.
- Limitar el acceso a datos sensibles mediante autenticación y permisos.

---

# Resumen

- Las **Custom Tools** permiten extender un agente con funcionalidades específicas creadas por el desarrollador.
- Pueden implementarse mediante **Azure Functions**, **Azure Logic Apps** o **APIs REST**.
- El agente decide automáticamente cuándo utilizar una herramienta y emplea el resultado para generar una respuesta.
- Son fundamentales para integrar agentes de IA con aplicaciones y procesos empresariales.
