# Crear un Azure AI Agent con Microsoft Agent Framework

## ¿Qué es un Microsoft Foundry Agent?

Un **Microsoft Foundry Agent** es un agente especializado del **Microsoft Agent Framework** diseñado para aplicaciones empresariales.

Proporciona de forma integrada:

- Gestión automática de conversaciones.
- Llamadas automáticas a herramientas (*Tool Calling*).
- Memoria mediante **Threads**.
- Integración con herramientas y servicios de Azure.

No es necesario implementar manualmente:

- Function Calling.
- Gestión del historial.
- Estado de la conversación.

---

# Capacidades principales

Un Foundry Agent incluye:

- Planificación (*Planning*).
- Memoria (*Memory*).
- Ejecución de funciones.
- Integración con herramientas.
- Seguridad empresarial.
- Gestión automática del contexto.

---

# Flujo de creación

Para crear un agente:

1. Crear un proyecto en Microsoft Foundry.
2. Obtener el **Connection String**.
3. Configurar la autenticación (`AzureCliCredential` u otra identidad de Azure).
4. Crear un `AzureOpenAIResponsesClient`.
5. Crear el `Agent` indicando:
   - instrucciones (*instructions*)
   - herramientas (*tools*)
6. Crear una sesión de conversación para interactuar con el agente.

```text
Microsoft Foundry Project
            │
            ▼
AzureOpenAIResponsesClient
            │
            ▼
Agent
            │
            ▼
Conversation (AgentSession)
            │
            ▼
Usuario
```

# Componentes principales

## AzureOpenAIResponsesClient

Gestiona la conexión con el proyecto de Microsoft Foundry.

Responsabilidades:

- Autenticación.
- Acceso a modelos.
- Acceso a herramientas.
- Seguridad empresarial.

---

## Agent

Es la clase principal del agente.

Combina:

- Cliente.
- Instrucciones.
- Herramientas.

Es quien ejecuta el razonamiento y responde al usuario.

---

## AgentSession

Gestiona automáticamente:

- Historial de conversación.
- Estado.
- Contexto.
- Threads.

Puede crear nuevos threads o reutilizar existentes.

---

## Tools Integration

Permite conectar el agente con:

- Azure AI Search
- Azure Functions
- APIs externas
- Funciones personalizadas

---

## Authentication

Soporta distintos mecanismos de autenticación, por ejemplo:

- Azure CLI Credential
- Service Principal
- Azure Identity

---

## Thread Management

Permite:

- Crear conversaciones nuevas.
- Mantener conversaciones existentes.
- Reutilizar contexto entre interacciones.

# Herramientas (Tools)

Los **Tools** permiten que el agente interactúe con servicios externos mediante **Function Calling**.

El agente decide automáticamente cuándo utilizar una herramienta.

```text
Usuario
      │
      ▼
Agente
      │
¿Necesito una Tool?
      │
 ┌────┴─────┐
 │          │
No         Sí
 │          │
 ▼          ▼
Responder   Ejecutar Tool
                │
                ▼
           Resultado
                │
                ▼
         Respuesta final
```

---

# Tipos de herramientas

## 1. Built-in Tools

Microsoft Foundry incluye herramientas listas para usar.

### Code Interpreter

Ejecuta código Python.

Usos:

- Cálculos.
- Estadística.
- Ciencia de datos.
- Manipulación de archivos.

---

### File Search

Permite buscar información dentro de documentos.

Usos:

- PDFs.
- Word.
- Manuales.
- Bases de conocimiento.

---

### Web Search

Consulta información actualizada en Internet.

Usos:

- Noticias.
- Datos recientes.
- Información externa.

# Custom Function Tools

También es posible crear herramientas propias.

## Crear una Tool

Se define una función Python utilizando el decorador:

```python
@tool
```

Este decorador registra automáticamente la función como una herramienta disponible para el agente.

---

## Describir correctamente la función

La IA necesita comprender:

- Qué hace.
- Cuándo utilizarla.
- Qué parámetros recibe.
- Qué devuelve.

Para ello se recomienda utilizar:

- Type Hints.
- `Annotated`.
- `Field` de Pydantic.

Mientras más descriptiva sea la función, mejor decidirá el agente cuándo utilizarla.

---

## Registrar herramientas

Las herramientas se agregan al crear el agente mediante el parámetro:

```python
tools=[...]
```

Puede registrarse:

- Una sola Tool.
- Varias Tools.

---

## Invocación automática

Una vez registradas:

No es necesario llamarlas manualmente.

El agente decide automáticamente:

- si utilizarlas,
- cuál utilizar,
- en qué momento hacerlo.

---

## Múltiples herramientas

Un agente puede disponer de varias herramientas.

El framework se encarga de:

- seleccionar la adecuada,
- ejecutarla,
- combinar resultados,
- generar la respuesta final.

---

# Buenas prácticas

Microsoft recomienda:

### Descripciones claras

Explicar claramente el propósito de la función y sus parámetros.

---

### Type Annotations

Definir correctamente:

- tipos de entrada,
- tipos de salida.

---

### Manejo de errores

Implementar validaciones y tratamiento de excepciones.

---

### Retornar información útil

Las funciones deben devolver datos que el modelo pueda utilizar para construir una buena respuesta.

---

### Funciones pequeñas

Cada Tool debería realizar una única tarea específica.

Es preferible muchas herramientas pequeñas que una única herramienta con demasiadas responsabilidades.

---

# Resumen

- Un **Microsoft Foundry Agent** es un agente empresarial que incorpora planificación, memoria, gestión de conversaciones y herramientas de forma integrada.
- El **AzureOpenAIResponsesClient** conecta la aplicación con Microsoft Foundry, mientras que **AgentSession** administra automáticamente el historial y el contexto.
- Los agentes pueden utilizar **Built-in Tools** (Code Interpreter, File Search y Web Search) o **Custom Function Tools** creadas mediante el decorador `@tool`.
- El framework se encarga del **Function Calling** y de la orquestación de herramientas, por lo que el desarrollador solo necesita definir funciones bien documentadas y agregarlas al agente.
- Para obtener mejores resultados, las herramientas deben ser específicas, contar con descripciones claras, utilizar anotaciones de tipos y manejar adecuadamente los errores.
