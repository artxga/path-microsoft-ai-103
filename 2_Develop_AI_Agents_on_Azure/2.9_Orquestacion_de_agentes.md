# Orquestación de Agentes (Agent Orchestration)

## ¿Qué es?

La **orquestación de agentes** permite coordinar múltiples agentes especializados para resolver tareas complejas de forma colaborativa.

En lugar de depender de un único agente, cada agente tiene una responsabilidad específica y trabaja junto con los demás.

```text
                Solicitud
                    │
                    ▼
        ┌─────────────────────┐
        │  Orquestador (SDK)  │
        └─────────────────────┘
          │        │        │
          ▼        ▼        ▼
     Agente A  Agente B  Agente C
          │        │        │
          └────────┴────────┘
                   ▼
           Respuesta final
```

El **Microsoft Agent Framework SDK** abstrae la coordinación entre agentes, permitiendo enfocarse en la lógica del negocio.

---

# ¿Por qué utilizar múltiples agentes?

Un único agente suele estar limitado por:

- Un solo conjunto de instrucciones.
- Un único contexto.
- Un único rol.

La orquestación permite:

- Asignar responsabilidades distintas a cada agente.
- Combinar múltiples perspectivas.
- Ejecutar procesos por etapas.
- Transferir tareas entre agentes según el contexto.
- Escalar soluciones complejas.

---

# Workflows en Microsoft Agent Framework

La orquestación se implementa mediante **Workflows**, que definen cómo interactúan agentes y componentes de lógica.

Los workflows permiten:

- Coordinar múltiples agentes.
- Controlar el flujo de ejecución.
- Guardar y reanudar el estado (*checkpointing*).
- Automatizar procesos complejos.

---

# Componentes de un Workflow

## Executors

Los **Executors** son los encargados de ejecutar cada paso del workflow.

Pueden representar:

- Un agente de IA.
- Un componente de lógica personalizada.

Ejemplo:

```text
Executor
    │
Analiza solicitud
    ▼
Resultado
```

---

## Edges

Los **Edges** determinan cómo fluye la información entre los executors.

### Direct Edge

Conecta un executor con el siguiente.

```text
Executor A
      │
      ▼
Executor B
```

Ideal para procesos secuenciales.

---

### Conditional Edge

Solo se ejecuta cuando se cumple una condición.

```text
          ¿Condición?
          /        \
        Sí          No
        │            │
        ▼            ▼
 Executor A    Executor B
```

Ideal para reglas de negocio.

---

### Switch-Case Edge

Selecciona un camino entre varios posibles.

```text
          Tipo
            │
   ┌────────┼────────┐
   ▼        ▼        ▼
Caso A   Caso B   Caso C
```

Ideal para clasificación o enrutamiento.

---

### Fan-Out Edge

Envía la misma solicitud a varios executors simultáneamente.

```text
          Entrada
             │
     ┌───────┼────────┐
     ▼       ▼        ▼
Exec A   Exec B   Exec C
```

Ideal para procesamiento paralelo.

---

### Fan-In Edge

Combina resultados provenientes de varios executors.

```text
Exec A
    │
Exec B
    ├────► Resultado final
Exec C
```

Ideal para consolidar información.

---

# Eventos del Workflow

Durante la ejecución, el framework genera eventos que facilitan el monitoreo y la depuración.

| Evento | Descripción |
|---------|-------------|
| **WorkflowStartedEvent** | Inicio del workflow |
| **WorkflowOutputEvent** | El workflow produjo una salida |
| **WorkflowErrorEvent** | Ocurrió un error |
| **ExecutorInvokeEvent** | Un executor comenzó su trabajo |
| **ExecutorCompleteEvent** | Un executor finalizó |
| **RequestInfoEvent** | Se realizó una solicitud externa |

Estos eventos mejoran la observabilidad y el diagnóstico.

# Patrones de orquestación

## 1. Concurrent Orchestration

Todos los agentes trabajan en paralelo sobre la misma tarea.

```text
             Solicitud
                 │
        ┌────────┼────────┐
        ▼        ▼        ▼
   Agente A Agente B Agente C
        │        │        │
        └────────┴────────┘
                 ▼
             Resultado
```

### Casos de uso

- Análisis paralelo.
- Ensemble de modelos.
- Subtareas independientes.

---

## 2. Sequential Orchestration

Cada agente procesa el resultado del anterior.

```text
Agente A
    │
    ▼
Agente B
    │
    ▼
Agente C
```

### Casos de uso

- Pipelines.
- Refinamiento progresivo.
- Procesos paso a paso.

---

## 3. Handoff Orchestration

El control pasa dinámicamente entre agentes según reglas o contexto.

```text
Usuario
    │
    ▼
Agente General
    │
 ¿Necesita experto?
    │
   Sí
    ▼
Agente Especialista
```

### Casos de uso

- Escalamiento.
- Enrutamiento inteligente.
- Fallback.

---

## 4. Group Chat Orchestration

Varios agentes participan en una conversación compartida.

Un **Chat Manager** decide quién interviene en cada momento.

```text
           Chat Manager
                │
      ┌─────────┼─────────┐
      ▼         ▼         ▼
 Agente A  Agente B  Agente C
```

### Casos de uso

- Brainstorming.
- Colaboración.
- Resolución conjunta de problemas.

---

## 5. Magentic Orchestration

Un **Manager Agent** planifica, delega y adapta la ejecución dinámicamente.

```text
        Manager Agent
              │
      ┌───────┼────────┐
      ▼       ▼        ▼
 Agente A Agente B Agente C
```

### Casos de uso

- Problemas abiertos.
- Procesos dinámicos.
- Delegación inteligente.

---

# Flujo unificado

Independientemente del patrón elegido, el proceso es similar:

```text
Definir agentes
        │
        ▼
Seleccionar patrón de orquestación
        │
        ▼
Configurar callbacks (opcional)
        │
        ▼
Iniciar runtime
        │
        ▼
Ejecutar tarea
        │
        ▼
Obtener resultados
```

Todos los patrones comparten la misma interfaz del SDK, facilitando cambiar de estrategia sin modificar la lógica de los agentes.

---

# Comparación de patrones

| Patrón | Ejecución | Mejor para |
|----------|-----------|------------|
| **Concurrent** | Paralela | Análisis independientes |
| **Sequential** | Lineal | Pipelines y procesos por etapas |
| **Handoff** | Dinámica | Escalamiento y especialistas |
| **Group Chat** | Conversacional | Colaboración entre agentes |
| **Magentic** | Manager + delegación | Problemas complejos y abiertos |

---

# Resumen

- La **orquestación de agentes** coordina múltiples agentes especializados para resolver tareas complejas de forma colaborativa.
- Los **Workflows** del Microsoft Agent Framework organizan la interacción entre agentes mediante **Executors** y **Edges**, además de proporcionar eventos para monitoreo y depuración.
- El SDK ofrece cinco patrones principales de orquestación:
  - **Concurrent:** ejecución paralela.
  - **Sequential:** procesamiento por etapas.
  - **Handoff:** transferencia dinámica entre agentes.
  - **Group Chat:** conversación colaborativa.
  - **Magentic:** un agente administrador planifica y coordina a los demás.
- Todos los patrones comparten una interfaz común, lo que permite experimentar con distintas estrategias sin reescribir la lógica de los agentes.
