# Mantenimiento de Workflows en Microsoft Foundry

## ¿Por qué es importante?

Crear un workflow es solo el primer paso. Con el tiempo, los procesos de negocio, los modelos de IA y los requisitos cambian, por lo que es necesario **mantener y refinar los workflows** para que sigan siendo:

- Confiables.
- Comprensibles.
- Adaptables.
- Fáciles de mantener.

Microsoft Foundry proporciona herramientas para facilitar esta tarea mediante **versionado**, **edición en YAML** y **documentación integrada**.

---

# Representaciones del Workflow

Los workflows pueden visualizarse de dos formas.

## 1. Visual Canvas

Representación gráfica mediante nodos.

Ideal para:

- Comprender el flujo.
- Visualizar la ejecución.
- Colaborar con otros miembros del equipo.

---

## 2. YAML

Representación textual del workflow.

Permite:

- Configuración avanzada.
- Integración con control de versiones (Git).
- Seguimiento de cambios.
- Edición manual.

> **Importante:** Ambas representaciones están sincronizadas. Los cambios realizados en el editor visual se reflejan en el YAML y viceversa.

---

# Versionado

Cada vez que se guarda un workflow, Microsoft Foundry crea automáticamente una **nueva versión inmutable**.

## Beneficios

- Consultar versiones anteriores.
- Comparar cambios.
- Restaurar una versión previa (*rollback*).
- Facilitar el trabajo colaborativo.
- Registrar quién realizó cada modificación y por qué.

El versionado actúa como una red de seguridad ante cambios que puedan introducir errores.

---

# Notas para mantenedores

El editor visual permite agregar **notas** a nodos o secciones del workflow.

Las notas pueden utilizarse para:

- Explicar decisiones de diseño.
- Documentar la lógica del flujo.
- Describir variables importantes.
- Proporcionar contexto a futuros desarrolladores.

Una buena documentación facilita el mantenimiento y reduce errores.

---

# Buenas prácticas

Microsoft recomienda:

- Eliminar nodos innecesarios o redundantes.
- Verificar que las salidas estructuradas de los agentes se procesen correctamente.
- Documentar la lógica mediante notas.
- Aprovechar el historial de versiones para revisar y validar cambios antes de publicarlos.

---

# Flujo de mantenimiento

```text
Modificar Workflow
        │
        ▼
Guardar
        │
        ▼
Nueva versión automática
        │
        ├── Comparar versiones
        ├── Restaurar versiones
        └── Documentar cambios
                │
                ▼
Workflow actualizado
```

---

# Resumen

- Los workflows pueden editarse tanto desde un **editor visual** como mediante **YAML**, manteniendo ambas vistas sincronizadas.
- Cada guardado genera automáticamente una **nueva versión**, lo que permite comparar cambios y restaurar versiones anteriores.
- Es recomendable documentar el workflow mediante **notas** para facilitar su mantenimiento.
- Mantener un workflow implica no solo corregir errores, sino también mejorar su claridad, eficiencia y capacidad de adaptación mediante buenas prácticas y control de versiones.
