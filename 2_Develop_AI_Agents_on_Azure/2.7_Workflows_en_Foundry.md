# Workflows en Microsoft Foundry

## ¿Qué son?

Los **Workflows** en Microsoft Foundry permiten **orquestar procesos impulsados por IA** mediante un enfoque **visual y declarativo**, sin necesidad de escribir código.

En lugar de implementar la lógica manualmente, se define una secuencia de **nodos** que representan las acciones que deben ejecutarse y el orden en que ocurren.

Foundry administra automáticamente:

- La ejecución del flujo.
- El estado del proceso.
- La transición entre pasos.

Son ideales para automatizar procesos de negocio que combinan:

- Agentes de IA.
- Lógica de negocio.
- Toma de decisiones.
- Interacción con usuarios.

---

# Componentes de un Workflow

Un workflow está compuesto por **nodos conectados**, donde cada nodo realiza una función específica.

Algunos ejemplos de nodos son:

- Invocar un agente.
- Evaluar condiciones.
- Manipular datos.
- Comunicarse con usuarios.
- Solicitar intervención humana.

```text
Inicio
   │
   ▼
[Nodo]
   │
   ▼
[Nodo]
   │
   ▼
[Nodo]
   │
   ▼
Fin
```

La forma en que se conectan los nodos determina el flujo de ejecución.

---

# ¿Por qué utilizar Workflows?

Los workflows permiten:

- Automatizar procesos complejos.
- Coordinar múltiples agentes especializados.
- Combinar IA con reglas de negocio.
- Incorporar supervisión humana cuando sea necesario.
- Escalar soluciones de IA de forma organizada.

Mientras que un único agente puede ser suficiente para tareas simples, los workflows permiten dividir responsabilidades entre varios agentes y coordinar su colaboración.

---

# Patrones de Workflow

Microsoft Foundry proporciona varios patrones predefinidos para estructurar la interacción entre agentes.

## 1. Sequential Workflow

Ejecuta cada paso en un orden fijo.

```text
Entrada
   │
   ▼
Validar datos
   │
   ▼
Enriquecer información
   │
   ▼
Generar respuesta
```

### Características

- Flujo lineal.
- Cada nodo recibe la salida del anterior.
- Fácil de entender y mantener.
- Comportamiento predecible.

### Casos de uso

- Procesamiento por etapas.
- Validación de datos.
- Generación de respuestas.
- Pipelines de automatización.

---

## 2. Human-in-the-loop Workflow

Introduce pausas para solicitar intervención humana antes de continuar.

```text
Proceso
   │
   ▼
Solicitar aprobación
   │
 ┌─┴────────┐
 │          │
Sí         No
 │          │
 ▼          ▼
Continuar  Finalizar
```

### Características

- Solicita aprobación o información adicional.
- Espera la respuesta del usuario.
- Continúa el flujo según la decisión tomada.

### Casos de uso

- Aprobaciones.
- Confirmaciones.
- Escalamiento de decisiones.
- Casos donde la IA necesita más contexto.

---

## 3. Group Chat Workflow

Permite la colaboración entre varios agentes especializados.

```text
Usuario
    │
    ▼
Agente Clasificador
    │
    ▼
Agente Especialista
    │
    ▼
Agente Resolutor
    │
    ▼
Respuesta
```

A diferencia del flujo secuencial, el control puede cambiar entre agentes dependiendo del contexto y de los resultados obtenidos.

### Características

- Colaboración entre agentes.
- Flujo dinámico.
- Los agentes pueden construir sobre las respuestas de otros.
- Mayor flexibilidad para tareas complejas.

### Casos de uso

- Soporte al cliente.
- Resolución de consultas multidominio.
- Automatización de procesos complejos.
- Sistemas con múltiples especialistas.

---

# Comparación de patrones

| Patrón | Flujo | Intervención humana | Escenario ideal |
|---------|-------|---------------------|-----------------|
| **Sequential** | Lineal | No | Procesos paso a paso |
| **Human-in-the-loop** | Lineal con pausas | Sí | Aprobaciones y supervisión |
| **Group Chat** | Dinámico | Opcional | Colaboración entre múltiples agentes |

---

# ¿Cómo elegir un patrón?

### Sequential

Utilízalo cuando:

- El proceso tenga pasos bien definidos.
- Cada etapa dependa de la anterior.
- Se requiera un flujo sencillo y predecible.

---

### Human-in-the-loop

Utilízalo cuando:

- Se necesite aprobación humana.
- Existan decisiones críticas.
- Haya información que solo una persona pueda proporcionar.

---

### Group Chat

Utilízalo cuando:

- Participen varios agentes especializados.
- La consulta requiera diferentes conocimientos.
- El flujo de trabajo no pueda definirse de forma estrictamente lineal.

---

# Resumen

- Un **Workflow** es una forma visual de orquestar procesos de IA mediante nodos conectados.
- Permite combinar agentes, lógica de negocio y participación humana en un mismo flujo.
- Microsoft Foundry ofrece tres patrones principales:
  - **Sequential:** flujo lineal y predecible.
  - **Human-in-the-loop:** incorpora intervención humana.
  - **Group Chat:** coordina múltiples agentes especializados.
- Elegir el patrón adecuado depende del nivel de automatización, complejidad y colaboración que requiera el proceso.
