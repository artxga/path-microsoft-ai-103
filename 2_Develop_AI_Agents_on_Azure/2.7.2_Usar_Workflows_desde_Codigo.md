# Usar Workflows desde código

## ¿Por qué invocar un Workflow desde código?

Después de diseñar y probar un workflow en **Microsoft Foundry**, puedes integrarlo en tus aplicaciones mediante el **Azure AI Projects SDK**.

Esto permite utilizar workflows desde:

- Aplicaciones web.
- APIs REST.
- Microservicios.
- Servicios backend.
- Procesos batch.

---

# Flujo general

Los workflows se crean visualmente en Microsoft Foundry y se almacenan como una definición **YAML**.

Desde código solo necesitas hacer referencia al nombre del workflow para ejecutarlo.

```text
Visual Designer
        │
        ▼
Workflow (YAML)
        │
        ▼
Azure AI Projects SDK
        │
        ▼
Aplicación
```

---

# Invocar un Workflow

Antes de ejecutar un workflow debes:

1. Conectarte al proyecto de Microsoft Foundry mediante **AIProjectClient**.
2. Crear una conversación.
3. Invocar el workflow utilizando su nombre.

Ejemplo:

```python
workflow_name = "triage-workflow"

conversation = openai_client.conversations.create()

stream = openai_client.responses.create(
    conversation=conversation.id,
    extra_body={
        "agent": {
            "name": workflow_name,
            "type": "agent_reference"
        }
    },
    input="Users can't reset their password from the mobile app.",
    stream=True
)
```

---

# Entrada del Workflow

El parámetro **input** proporciona la información inicial que utilizará el workflow.

Dependiendo del diseño, puede representar:

- Una pregunta del usuario.
- La descripción de un ticket de soporte.
- Un conjunto de datos para procesar.
- Una cadena vacía para iniciar el flujo sin entrada específica.

---

# Procesar eventos del Workflow

Si la ejecución utiliza **streaming**, la aplicación recibe eventos en tiempo real mientras el workflow se ejecuta.

Ejemplo:

```python
for event in stream:
    if event.type == "response.completed":
        ...

    if event.type == "response.output_item.done":
        ...
```

Esto permite mostrar el progreso y reaccionar a cada paso del workflow.

---

# Eventos principales

| Evento | Descripción |
|---------|-------------|
| **response.completed** | El workflow terminó y devolvió la respuesta final. |
| **response.output_item.done** | Finalizó una acción individual del workflow. |

---

# Workflows con Human-in-the-loop

Si el workflow requiere intervención humana:

1. El flujo se pausa.
2. La aplicación solicita información al usuario.
3. El usuario responde.
4. La conversación continúa enviando un nuevo mensaje.
5. El workflow reanuda su ejecución.

```text
Workflow
     │
     ▼
Solicita información
     │
     ▼
Usuario responde
     │
     ▼
Workflow continúa
```

# Beneficios de la integración mediante código

| Escenario | Beneficio |
|------------|-----------|
| **Aplicaciones web** | Incorporar workflows directamente en la interfaz de usuario. |
| **APIs y microservicios** | Exponer workflows mediante endpoints REST. |
| **Procesamiento batch** | Ejecutar workflows sobre grandes volúmenes de datos. |
| **Pruebas automatizadas** | Integrar workflows en pipelines de CI/CD. |
| **Interfaces personalizadas** | Crear experiencias adaptadas al flujo de trabajo del negocio. |

# Flujo completo

```text
Workflow diseñado en Foundry
            │
            ▼
Guardar en el proyecto
            │
            ▼
Invocar desde Azure AI Projects SDK
            │
            ▼
Crear conversación
            │
            ▼
Enviar input
            │
            ▼
Procesar eventos
            │
            ▼
Obtener respuesta
```

---

# Resumen

- Los workflows creados en Microsoft Foundry pueden ejecutarse desde código mediante el **Azure AI Projects SDK**.
- La aplicación invoca un workflow haciendo referencia a su **nombre** dentro del proyecto.
- El parámetro **input** proporciona la información que inicia la ejecución.
- Con **streaming** es posible recibir eventos en tiempo real, como el progreso del workflow o la finalización de acciones individuales.
- Integrar workflows en código permite incorporarlos fácilmente en aplicaciones, APIs, procesos batch y pipelines de automatización.
