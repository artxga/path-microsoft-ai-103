# Foundry IQ

## ¿Qué es?

**Foundry IQ** es un servicio de **Microsoft AI Foundry** que permite crear **bases de conocimiento reutilizables** para que múltiples agentes de IA puedan acceder a información empresarial de forma segura y fundamentada.

Su objetivo es resolver el problema del conocimiento en los agentes, conectándolos con datos internos y externos en tiempo real mediante **Retrieval-Augmented Generation (RAG)**. :contentReference[oaicite:0]{index=0}

---

# Problema que resuelve

Los modelos de lenguaje tienen limitaciones:

- Su conocimiento tiene una fecha de corte.
- No pueden acceder a información privada de una organización.
- Pueden generar respuestas incorrectas o alucinaciones.

Foundry IQ permite que los agentes consulten una **base de conocimiento compartida** antes de responder. :contentReference[oaicite:1]{index=1}

---

# Componentes principales

## 1. Knowledge Sources

Son las fuentes de datos que contienen la información.

Ejemplos:

- Azure AI Search
- Azure Blob Storage
- SharePoint
- OneLake
- Sitios web públicos

Una base de conocimiento puede conectarse a múltiples fuentes. :contentReference[oaicite:2]{index=2}

---

## 2. Knowledge Base

Es el componente principal de Foundry IQ.

Una **Knowledge Base** agrupa una o varias **Knowledge Sources** y define cómo se recuperará la información.

Características:

- Reutilizable por múltiples agentes.
- Configurable.
- Punto único de consulta.

---

## 3. Agentic Retrieval

Es el motor inteligente de recuperación de información.

No se limita a buscar documentos; también puede:

- Descomponer consultas complejas.
- Realizar varias búsquedas en paralelo.
- Combinar resultados.
- Reordenar la información por relevancia.
- Devolver respuestas fundamentadas con citas. :contentReference[oaicite:3]{index=3}

---

# ¿Cómo funciona?

```text
Usuario
      │
      ▼
Agente
      │
      ▼
Knowledge Base
      │
      ▼
Knowledge Sources
(SharePoint, Blob, OneLake, Web...)
      │
      ▼
Agentic Retrieval
      │
      ▼
Información relevante
      │
      ▼
Respuesta con citas
```

---

# Flujo de trabajo

1. Crear una **Knowledge Source**.
2. Crear una **Knowledge Base** que utilice una o más fuentes.
3. Conectar la Knowledge Base al agente.
4. El agente consulta la base cuando recibe una pregunta.
5. Foundry IQ recupera la información y devuelve una respuesta fundamentada. :contentReference[oaicite:4]{index=4}

---

# Capacidades

Foundry IQ permite:

- Compartir una misma base de conocimiento entre varios agentes.
- Conectarse a múltiples orígenes de datos.
- Automatizar la creación de embeddings y el *chunking* de documentos.
- Ejecutar búsquedas por palabras clave, vectoriales o híbridas.
- Aplicar permisos de acceso según el usuario.
- Generar respuestas con referencias a las fuentes. :contentReference[oaicite:5]{index=5}

---

# Beneficios

- Centraliza el conocimiento empresarial.
- Reduce las alucinaciones.
- Facilita reutilizar la misma información en distintos agentes.
- Mantiene los permisos de acceso a los datos.
- Evita implementar lógica de recuperación para cada aplicación.

---

# Casos de uso

- Asistentes internos para empleados.
- Chatbots empresariales.
- Soporte técnico basado en documentación.
- Consulta de políticas y procedimientos.
- Agentes que necesitan acceder a información distribuida en múltiples sistemas.

---

# Foundry IQ vs RAG tradicional

| RAG tradicional | Foundry IQ |
|-----------------|------------|
| Cada aplicación implementa su propia lógica de recuperación. | La recuperación está centralizada en una **Knowledge Base** reutilizable. |
| Normalmente utiliza una única fuente de datos. | Puede combinar múltiples fuentes de datos. |
| La lógica suele desarrollarse manualmente. | Foundry IQ administra el proceso de recuperación (*Agentic Retrieval*). |
| Difícil de reutilizar entre aplicaciones. | Una misma Knowledge Base puede ser utilizada por varios agentes. |

---

# Resumen

- **Foundry IQ** es una plataforma de conocimiento compartido para agentes de IA.
- Utiliza **Knowledge Sources** para conectarse a distintas fuentes de información.
- Agrupa esas fuentes en una **Knowledge Base** reutilizable.
- Emplea **Agentic Retrieval** para recuperar información relevante y generar respuestas fundamentadas.
- Permite que múltiples agentes compartan el mismo conocimiento, reduciendo alucinaciones y respetando los permisos de acceso a los datos. :contentReference[oaicite:6]{index=6}
